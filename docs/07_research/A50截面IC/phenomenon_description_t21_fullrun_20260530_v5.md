#### 窗口数量口径说明（v5修正）

验证脚本 `validate_golden_baseline.py` 加载数据后，按以下逻辑处理：

| 概念 | 数值 | 口径说明 |
|:---:|:---:|:---:|

| momentum_20d有效窗口数 | 415个 | 431个截面中 rank_ic NOT NULL 的窗口数，即 `compute_baseline_metrics()` 中 `rank_ic.dropna()` 后的长度 |

**计算机制：**
- 验证脚本并非采用传统滑动窗口（指定窗口大小+步长）的滚动计算方式
- 431 = `len(ic_series)` = DB中所有 factor_name='momentum_20d' 且在指定日期范围内的截面总数
- 415 = `len(rank_ic.dropna())` = 上述431个截面中 rank_ic 非空的截面数（16个截面 rank_ic 为NULL，被排除）
- 窗口数（window count）在此语境下是截面数（数目的同义概念），而非滑动窗口划分的产物

**关键区别：** 431个"总窗口"（total_windows）和415个"有效窗口"（n_windows）均来源于验证脚本对 DB 数据的读取口径，是 momentum_20d 因子独有的截面计数，而非全因子总截面（610个）。三个数字（610截面 / 431总窗 / 415有效窗）的关系为：**全因子截面总数 → momentum_20d因子截面数 → 有效计算截面数**。

> ⚠️ **v4→v5 口径更正：** v4中将"415"同时作为"有效截面数"和"有效窗口数"使用，实际两者对应不同概念。v5中明确区分：
> - **全因子 rank_ic NOT NULL截面数：** ~426个（18:12快照，跨所有因子）
> - **momentum_20d有效窗口数：** 415个（仅针对momentum_20d因子）

## 问题间关联说明（Owner 18:59评估要求补充）

三个问题在当前现象描述中被独立并列呈现，但它们之间存在依赖关系，需要在此明确说明：

### 核心依赖链

```
问题1（SIGKILL/OOM）
  ↓ 影响
问题2（黄金基线FAIL）—— 数据基础来源
  ↓ 独立
问题3（估值因子IC=0）—— 数据源问题，与管线运行无关
```

### 依赖关系详解

**问题1 → 问题2：问题2的415个有效窗口的数据基础**
- 问题2基线验证使用的数据（momentum_20d的431总截面/415有效窗口）中：
  - 第1次（marine-s）产出222截面 → 通过 INSERT OR IGNORE 写入 a50_cross_ic_result 表
  - 第2次（mellow-o）产出428截面（自动跳过 marine-s 已有的222截面）→ 追加写入同一表
  - 第3次（--no-run 验证）读取的正是这两次写入合并在库中的数据
- **关键结论：** 问题2的数据基础不是"428截面"单次运行产物，而是两次独立运行数据的合并结果（去重后610个唯一截面）。**两次运行均被SIGKILL终止这一事实不影响数据完整性**，因为逐截面提交（INSERT OR IGNORE）保证每写完一个截面就持久化，进程被杀死时已完成的部分不会丢失
- **风险点（v5.1 修正）：** 此前的"写到哪算哪"表述不准确。实际全量数据库确认：
  - a50_cross_ic_result 全表最小日期 = 2007-01-05，**最大日期 = 2026-05-19**（全量范围终点）
  - 2026-05-19 之后**不存在任何截面**（0个日期）
  - 第2次运行（mellow-o）产出428截面的时间范围为 2007-01-05 ~ 2026-05-19，**中间无缺口**
  - **澄清：** mellow-o 在被杀时已完成了所有428截面的写库（数据完整覆盖全量范围），而非"写到一半被中断"。进程被杀死发生在**写库完成后的后处理阶段**（如IC序列聚合或报告生成），而非写库过程中。此修正不影响其他结论（数据自身无污染、逐截面提交保证写入原子性均成立）

**问题1 与 问题3：无依赖关系**
- 估值4因子的IC全零是数据源问题，与管线是否被SIGKILL杀死无关
- 即使管线完整运行，若 stock_daily 表的 pe/pb/pe_ttm 字段全部为NULL，估值因子的IC仍会是0或NULL
- 这两个问题可以并行处理，互不阻塞

**问题2 与 问题3：无依赖关系，但存在判定影响**
- 问题3的结果（估值因子全部无效）会扩大整体影响范围：**黄金基线FAIL + 估值因子全灭 = 候选因子池可用因子从18个减少至14个**
- 但估值因子的缺失不改变 momentum_20d 基线的判定结果（判定只基于 momentum_20d 一个因子）

### 综合判断

| 依赖关系 | 是否阻塞 | 说明 |
|:-------:|:-------:|:----|
| 问题1 → 问题2 | ⚠️ **有限影响** | SIGKILL导致第2次运行仅产出428截面（非全量），数据覆盖不完整；但已写入数据自身无污染，415个有效窗口的IC统计结果可信 |
| 问题1 → 问题3 | ❌ 无影响 | 独立问题，并行处理 |
| 问题2 ↔ 问题3 | ❌ 无影响 | 独立因子集，判定互不干扰 |
