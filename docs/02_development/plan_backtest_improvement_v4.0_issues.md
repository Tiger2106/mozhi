## 🟡 P1: 单文件被多项改动交叉修改

**发现者**: 主人 | **严重度**: P1 | **状态**: 待处理

### 问题描述
根据文件修改统计，以下策略文件被多项改动交叉命中：

| 文件 | 涉及项 | 改动处数 |
|:-----|:-------|:--------:|
| `run_grid.py` | 第1项(1c)、第2项(2c)、第3项(3b)、第4项(4a, 4d) | 5处 |
| `run_trend.py` | 第1项(1d)、第2项(2d)、第3项(3c)、第4项(4b, 4e) | 5处 |
| `run_reversal.py` | 第1项(1e)、第2项(2e)、第3项(3d)、第4项(4c, 4f) | 5处 |

P0和P1阶段反复打开同一文件，容易产生合并冲突或遗漏。

### 建议修复
- 对 `run_grid.py` / `run_trend.py` / `run_reversal.py` 的所有改动点汇总为一张变更表
- 开发前明确每个文件一次性的函数边界，避免逐项反复编辑相同区域
