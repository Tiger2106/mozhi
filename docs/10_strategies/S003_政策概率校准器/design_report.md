## §1 背景

### 1.1 策略动机
八部门整治政策涉及多个部门、多种工具，市场对每一政策信号的"落地概率"存在巨大分歧。一篇监管谈话可能被解读为"强信号"或"例行公事"——缺乏客观的概率校准，投资决策容易受舆论噪音放大。

现有方法要么依赖分析师主观概率，要么使用过于粗糙的"发生/不发生"二元判断。需要一个基于规则+历史数据双轨校准的系统，为每个政策信号赋予一个校准后的落地概率。

### 1.2 要解决的问题
- 政策信号噪音大：大量表态、谈话、窗口指导不一定会转化为正式文件或行动
- 概率判断不一致：不同Agent对同一政策信号的概率判断可能相差甚远
- 校准无反馈机制：历史预测概率未与真实结果建立闭环

### 1.3 与八部门整治政策的关系
S003 直接服务于八部门整治政策中所有可预判的政策信号。它不负责监测"堵疏力度"（那是S002的职责），而是回答"这条政策信号落地概率多大？"

---

## §2 设计目标

### 2.1 核心功能定位
建立政策信号校准概率模型，采用规则表+历史校准双轨机制，为每个政策信号提供校准后的落地概率。

### 2.2 预期产出

| 产出项 | 格式 | 说明 |
|:----|:----|:----|
| 政策信号概率表 | JSON | 当前每一个活跃政策信号的概率估算 |
| 规则触发记录 | JSON | 规则表中哪些规则被触发及依据 |
| 历史校准日志 | JSON | 每条信号概率与历史相似信号比较 |
| 校准反馈报告 | Markdown | 每月/每季的预测概率vs实际结果偏差分析 |

### 2.3 使用场景
- **信号触发后即时**：玄知检测到新政策信号时，S003立即输出概率
- **每日刷新**：对存量信号重新评估，条件变化可改变概率
- **月度校准**：回顾当月预测，调整规则表权重
- **Agent调用**：供墨衡（S001情景推演概率输入）和墨涵（投研报告支撑）调用

## §3 技术方案

### 3.1 核心算法：双轨校准

**Track A — 规则表校准**

```
输入：信号特征向量 (发文人级别, 措辞强度, 发布渠道, 历史同类落地率, ...)
   │
   ▼
规则引擎：布尔规则集
   │  (例：发文人=国务院副总理级 + 措辞="坚决打击" → 落地概率 ≥0.8)
   │  (例：发文人=普通官员 + 措辞="关注" → 落地概率 ≤0.3)
   │  (例：发布渠道=证监会公告 → 落地概率 ≥0.6)
   ▼
Track A 概率 ← 规则加权平均
```

**规则表结构**：

```json
{
  "rules": [
    {
      "id": "R001",
      "condition": "publisher_level == 'minister' AND severity >= 0.8",
      "prob_range": [0.7, 0.95],
      "weight": 2.0,
      "source": "2018-2025历史统计"
    },
    {
      "id": "R002",
      "condition": "channel == 'window_guidance'",
      "prob_range": [0.3, 0.6],
      "weight": 1.5,
      "source": "专家评议"
    }
  ]
}
```

**Track B — 历史校准**

```
输入：当前信号特征向量
   │
   ▼
向量化 → 余弦相似度匹配历史信号库
   │  (历史信号库包含：信号特征、最终是否落地、落地时间差)
   ▼
匹配历史信号集 → 统计该特征区间信号的落地率
   │  (最低相似度阈值 0.7)
   ▼
Track B 概率 ← 历史相似信号落地率
```

**融合机制**：
- Track A 权重 w_A 固定为 0.6（规则可靠优先）
- Track B 权重 w_B 由匹配度动态调整（0.1~0.4），匹配数越多权重越高
- 最终概率 P = w_A × P_A + w_B × P_B
- 当两条概率偏差超过 0.3 时，触发"分歧标注"，输出两条概率而非融合结果

### 3.2 信号触发条件

S003 不主动扫描政策信息。由玄知引擎在检测到以下事件时主动推送信号：

| 触发条件 | 示例 | 优先级 |
|:----|:----|:----:|
| 官方文件发布 | 公告、通知、办法 | 高 |
| 高级别讲话 | 部级以上官员公开发言 | 高 |
| 窗口指导确认 | 券商/银行内部传达确认 | 中 |
| 媒体独家报道 | 可信源报道政策动向 | 中 |
| 社交网络发酵 | 政策相关舆论热度超阈值 | 低（需交叉验证） |

### 3.3 校准反馈闭环

```
每月/每季：
  1. 收集上期所有预测的"落地/未落地"实际结果
  2. 计算校准误差 = 预测概率均值 - 实际落地率
  3. 校准误差 > 0.15 → 调整规则表权重
  4. 校准误差 > 0.25 → 增加/修改规则条目
  5. 输出校准报告
```

## §4 接口协议

### 4.1 输入接口

**接口名称**：`inputs/policy_signal`

由玄知在检测到新信号时调用。

| 字段 | 类型 | 必填 | 说明 |
|:----|:----|:----|:----|
| signal_id | string | 是 | 信号唯一标识 |
| publisher | string | 是 | 发布人/部门 |
| publisher_level | string | 是 | 发文人级别代码 |
| content_excerpt | string | 是 | 信号内容摘要 |
| channel | string | 是 | 发布渠道 |
| severity | float[0,1] | 是 | NLP措辞严重度评分 |
| timestamp | string | 是 | 信号检测时间(ISO8601) |
| source_type | string | 是 | 触发条件类型 |

### 4.2 输出接口

**接口名称**：`outputs/calibrated_probability`

| 字段 | 类型 | 说明 |
|:----|:----|:----|
| signal_id | string | 原信号ID |
| track_a_prob | float[0,1] | 规则表校准概率 |
| track_b_prob | float[0,1] | 历史校准概率 |
| final_prob | float[0,1] | 融合后最终概率 |
| confidence | enum[高,中,低] | 置信度 |
| divergence_flag | bool | 双轨是否分歧 |
| matched_history_count | int | 匹配历史信号数 |
| rule_triggered | string[] | 触发规则ID列表 |

### 4.3 上游依赖
- **玄知**：提供政策信号输入（独家输入源）
- **历史信号库**：由 S003 自维护，玄知辅助标注落地状态

### 4.4 下游消费者
- **S001**：政策限制折扣 α₁ 计算需要政策信号概率
- **墨涵**：投研报告引用政策信号概率

## §5 验证标准

### 5.1 通过率指标

| 指标 | 目标值 | 说明 |
|:----|:----|:----|
| 概率校准Brier得分 | ≤0.12 | 预测概率与真实结果的均方误差 |
| 规则表覆盖度 | ≥80% | 输入信号中规则表可覆盖的比例 |
| 双轨分歧率 | ≤15% | Track A/B 分歧占比 |
| 历史匹配率 | ≥60% | 输入信号中找到相似历史信号的比例 |
| 置信度校准 | 高/中/低三档的实际落地率与设定区间一致 |

### 5.2 交叉验证方法
- 按季度进行校准回测，使用上一季度数据预测->对比实际结果
- 引入"盲测"：对同一信号同时由 S003 和外部专家团队(如2位政策分析师)独立评估，比较 S003 与专家平均概率的偏差
- 规则表每半年由玄知人工审计一次，调整过期规则

## §6 风险与边界

### 6.1 已知局限性

| 风险 | 描述 | 应对 |
|:----|:----|:----|
| 规则老化 | 政策生态变化导致旧规则失效 | 季度规则审计 |
| 历史信号偏差 | 历史库中大量是2018-2025数据，新政策形态可能无相似案例 | 标注"低匹配" |
| 信号时效性 | 信号随时间推移概率衰减（未落地的信号概率下降） | 加入时间衰减因子 |
| 黑天鹅信号 | 完全超出规则的信号（如突发高规格闭门会议） | 设兜底规则：未知分类输出0.5 |

### 6.2 边界条件
- 当日无新信号输入时：不输出
- 单信号检测时置信度"低"但分歧标注触发：输出双轨概率，标注"请人工评估"
- 连续10条信号均无法匹配历史 → 触发规则库审计

### 6.3 熔断/降级路径
1. **Brier得分连续3月超过0.15** → 规则表整体权重降为原50%，更多依赖历史校准
2. **季度校准误差>0.3** → 冻结规则表，全部切换至Track B模式，同时启动规则重建
3. **Track B亦告警**（历史匹配率<30%） → 切换为纯规则模式，输出标记"低置信度"
