### knowledge.db 松耦合关系详解

```
┌─────────────────────────────────────────────────────────────────┐
│                       knowledge.db 数据模型                      │
│                                                                 │
│  ┌──────────────────────┐   ┌──────────────────────────────┐   │
│  │ strategy_ratings     │   │ parameter_recommendations    │   │
│  ├──────────────────────┤   ├──────────────────────────────┤   │
│  │ id: INTEGER PK       │   │ id: INTEGER PK               │   │
│  │ symbol: TEXT         │   │ strategy_id: INTEGER FK      │   │
│  │ strategy: TEXT       │   │ symbol: TEXT                 │   │
│  │ grade: TEXT (A/B/C/D)│   │ param_name: TEXT             │   │
│  │ total_return: REAL   │   │ optimal_value: REAL          │   │
│  │ sharpe: REAL         │   │ robust_range: TEXT           │   │
│  │ max_dd: REAL         │   │ confidence: REAL [0,1]       │   │
│  │ win_rate: REAL       │   │ backtest_hash: TEXT          │   │
│  │ sample_size: INT     │   │ created_at: ISO8601          │   │
│  │ backtest_hash: TEXT  │   │ expires_at: ISO8601          │   │
│  │ created_at: ISO8601  │   └──────────────────────────────┘   │
│  │ expires_at: ISO8601  │                                       │
│  └──────────────────────┘   ┌──────────────────────────────┐   │
│                              │ backtest_failures            │   │
│  ┌──────────────────────┐   ├──────────────────────────────┤   │
│  │ strategy_summaries   │   │ id: INTEGER PK               │   │
│  ├──────────────────────┤   │ symbol: TEXT                 │   │
│  │ id: INTEGER PK       │   │ strategy: TEXT               │   │
│  │ strategy_id: TEXT    │   │ failure_type: TEXT           │   │
│  │ summary: TEXT        │   │ description: TEXT            │   │
│  │ key_findings: TEXT   │   │ severity: TEXT (low/mid/high)│   │
│  │ tags: TEXT (JSON)    │   │ created_at: ISO8601          │   │
│  │ created_at: ISO8601  │   └──────────────────────────────┘   │
│  └──────────────────────┘                                       │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询接口（只读，供交易系统调用）                            │   │
│  │                                                         │   │
│  │  SELECT grade, sharpe, max_dd                           │   │
│  │  FROM strategy_ratings                                  │   │
│  │  WHERE symbol = '601857' AND strategy = 'trend'         │   │
│  │  ORDER BY created_at DESC LIMIT 1;                      │   │
│  │                                                         │   │
│  │  → 结果：墨衡分析时读回测经验，辅助信号判断               │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 写入接口（只写，供回测系统调用）                            │   │
│  │                                                         │   │
│  │  INSERT INTO strategy_ratings (...)                      │   │
│  │  VALUES ('601857', 'trend', 'A', 0.23, 1.8, 0.12, ...)  │   │
│  │                                                         │   │
│  │  → 回测系统运行结束后自动写入                              │   │
│  │  → 每次写入用 backtest_hash 去重，避免重复记录            │   │
│  │  → expires_at = 写入时间 + 90天 (策略有效性窗口)         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、物理隔离策略

### 文件系统隔离

```
回测系统文件空间:                   交易系统文件空间:
  mozhi_platform/                    mozhi_platform/
  ├── backtest_tools/               ├── reports/
  │   ├── engine/                   │   ├── morning/
  │   ├── strategies/               │   ├── evening/
  │   ├── param_sweep/              │   ├── daily/
  │   └── experiments/              │   └── ...
  ├── reports/research/             ├── signals/
  └── knowledge.db ← ------- 共享 --→ ├── triggers/
                                      ├── pipeline_complete/
                                      └── ...

  ❌ 不共享:
     - signals/triggers/ 目录
     - reports/morning/ 目录 (晨报)
     - reports/evening/ 目录 (结算)
     - trade_engine.db (交易数据库)
     - 所有 cron 任务

  ✅ 唯一共享: knowledge.db (只读/只写分离)
```

### 进程隔离

```
回测系统:                              交易系统:
  - 独立 Python 进程                      - 独立 Python + Node.js 进程
  - 无定时 cron (手动 + 按需)             - openclaw cron 驱动 (08:00~19:50)
  - 可随时启动/停止/重启                   - 固定时间线运行
  - 不影响交易系统运行                     - 不受回测系统影响
```

### 数据库隔离

```
交易系统数据库:                         回测系统数据库:
  trade_engine.db                       knowledge.db ←- 仅此共享
  file_registry.db                      市场行情数据 (market.db) - 可共享
  factor_repository.db
```

