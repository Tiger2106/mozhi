## 墨涵汇总：统一意见

### 决议
**两个因子全部放行，按以下方案实施：**

### 实施顺序（串行）
```
第1步（因子1）：新增成交额均价 → 0.5人天 → 墨衡执行 → 墨萱验收
      ↓
第2步（DB清理）：清空 market_daily.turnover_rate=0.0 的3080条历史记录
      ↓
第3步（因子2）：新增实际换手率 → 2.0人天 → 墨衡执行 → 墨萱验收
```

### 实施要点
1. **因子1**：走 `BaseFactor` 继承 + `FactorRegistry` 注册标准化路径，已完成数据验证
2. **因子2**：利用AKShare原生 `换手率` 字段 + `market_daily.turnover_rate` 列映射
3. **DB清理**：实施因子2前必须先清除历史占位数据
4. **因子2边界条件**：需标注换手率的 size effect（小盘股天然高换手）和次新股过滤（上市不足6个月的标的排除）
5. **Layer Q**：两个因子均不需新增Q维度，但换手率稳定性建议引入Q5参数
6. **知识库集成**：按现有 KnowledgeBridge 流程自动引入新因子值

### 预估总工时
- 因子1：0.5人天
- DB清理：0.2人天（含验证）
- 因子2：2.0人天（含测试）
- **总计：约2.7人天**
