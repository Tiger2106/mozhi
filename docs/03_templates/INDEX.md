# 墨枢报告模板库目录

## 使用说明

- 回测执行后，墨衡按对应模板填写报告
- 验证实验执行后，墨衡按VMPL-001模板填写验证报告
- 评审阶段，墨萱按QA清单模板独立验证
- 模板文件头部包含元信息 JSON，支持机器解析
- 原始工作副本位于 `mo_zhi_sharereports/templates/`

## 模板清单

| 编号 | 文件 | 类型 | 面向 | 责任人 | 大小 |
|:----:|:----|:----:|:----|:------:|:----:|
| TMPL-001 | `TMPL-001_backtest_summary_template.md` | 回测 | Owner/决策者 | 墨衡 | 6.2 KB |
| TMPL-002 | `TMPL-002_backtest_analysis_template.md` | 回测 | 评审方/技术人员 | 墨衡 | 10.6 KB |
| TMPL-003 | `TMPL-003_backtest_qa_checklist_template.md` | 回测 | QA独立验证 | 墨萱 | 8.0 KB |
| VMPL-001 | `VMPL-001_verification_template.md` | 验证 | 技术团队/执行方/QA | 墨衡 | 9.6 KB |

## 模板概览

### TMPL-001: 回测总结报告模板
输出方：墨衡（回测执行）

| 章节 | 内容 |
|:----|:-----|
| §1 核心结论 | 80字以内，含三档判定（实盘/观察/废弃）+ confidence |
| §2 核心指标 | 收益/回撤/夏普/胜率 vs 基准对比表 |
| §3 策略配置 | 入场/出场/仓位规则简要说明 |
| §4 执行方自检清单 | 引擎/参数/报告/数据库/Git版本/代码版本标注/环境信息/超时检查 |
| §5 风险评估 | 关键假设 + 影响分析 |
| §6 下一步建议 | 改进方向及优先级 |
| §7 审计日志 | 操作人+时间+操作类型 |

### TMPL-002: 回测分析报告模板
输出方：墨衡（回测执行），墨萱独立填写QA章节

| 章节 | 内容 |
|:----|:-----|
| §1 回测配置 | 引擎/参数/标的/代码版本(含无Git备选)/环境信息(OS/Python/依赖库)/超时策略(40min) |
| §2 绩效归因 | 总回报/年化收益/风险调整指标 |
| §3 交易统计 | 胜率/盈亏比/持仓分布/最大连续损益 |
| §4 市场环境敏感性 | 上涨/下跌/震荡市表现 |
| §5 参数敏感性 | 关键参数扫描结果 |
| §6 QA验证（墨萱填写） | 数据完整性/一致性/异常检测/复现性 |
| §7 审计日志 | 端到端审计链 |

### TMPL-003: 回测QA验证清单模板
输出方：墨萱（独立QA验证）

| 章节 | 内容 |
|:----|:-----|
| §1 验证元信息 | run_id、报告id、QA负责人 |
| §2 数据完整性 | 表行数校验、bar覆盖度、交易记录完整、数据库行数核验 |
| §3 数据一致性 | 6条明确标准（无前视偏差、复权一致等）+ 报告数据与数据库逐项交叉验证 |
| §4 异常检测 | 信号异常值、净值异常波动 |
| §5 复现性 | 同参数二次回测偏差≤2% + 确定性策略偏差0.00%标注说明 |
| §6 超时状态 | 实际耗时 vs 阈值 |
| §7 审计日志 | QA全程操作记录 |

### VMPL-001: 验证实验报告模板
输出方：墨衡（验证执行），墨萱独立填写QA章节（VQA）

| 章节 | 内容 |
|:----|:-----|
| §1 验证结论与核心指标 | 结论判定（PASS/FAIL/INCONCLUSIVE）+ 6项核心指标（通过率/一致性/边界通过率/模式覆盖率/复现性/执行有效性）|
| §2 验证配置 | 工具/被测对象/测试数据/验收标准版本 |
| §3 详细验证结果 | 逐标准详情 + 模式归因 + 敏感度分析 |
| §4 风险评估 | 覆盖不完整/实现偏差/环境依赖/边界遗漏/参数泛化 |
| §5 自检清单 | 8项检查（含代码版本/环境信息/种子锁定）|
| §6 审计日志 + 附录 | 审计链 + 代码版本/数据版本/环境信息 |

**QA接口：** 复用TMPL-003结构调整标准，VQA产出文件 `vqa_{task_id}.md`
