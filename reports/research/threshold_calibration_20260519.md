### 2.4 C4: 非单段收益（当前阈值: 最大单笔占比 < 40%）

| P报告 | 实际最大占比 | 当前判定 | 分析 |
|:----:|:-----------:|:--------:|:-----|
| P1(2笔) | ~98% | ❌ FAIL | 2笔中1盈1亏，盈者占98% |
| P2(7次回撤) | ~14% | ✅ PASS | 回撤均匀分布 |
| P3(66通过) | ~50% | ❌ FAIL | 部分参数收益集中 |
| P4(1有效窗格) | 100% | ❌ FAIL | 仅1窗格有交易 |
| P5(2笔) | ~98% | ❌ FAIL | 同P1 |
| P6-84(2笔) | ~100% | ❌ FAIL | 单笔交易占全部收益 |
| P6-3年(4笔) | ~50-60% | ❌ FAIL | 少量交易收益集中 |
| P7(1540天) | ~2-5% | ✅ PASS | 足够分散 |
| P8(2笔) | ~98% | ❌ FAIL | 同P1 |

**通过率**: 2/9 = 22.2%

**⚠️ 关键发现 —— C4 严重误杀**: 当 n ≤ 10 时，C4的40%阈值几乎必然不通过。这是因为绝对值和的计算方式使得"少数交易"天然集中。例如：
- n=2, 收益 +0.054% 和 -0.001% → 绝对值 = 0.054+0.001=0.055, max_share=0.054/0.055=98.2%
- n=3, 收益 +0.03%, +0.02%, -0.01% → 绝对值=0.06, max_share=0.03/0.06=50%

**这不是C4有效检测了"极端交易依赖"，而是C1不足时C4的连带失效。**

**建议**:
- **引入自适应修正**: `effective_max_share = max_share * sqrt(n / 30)` — 当 n < 30 时放宽阈值
  - n=2 → 有效阈值 = 40% × sqrt(2/30) = 40% × 0.258 = **10.3%** → 仍会FAIL（正确：2笔交易本就不该通过存在性验证）
  - n=10 → 有效阈值 = 40% × sqrt(10/30) = 40% × 0.577 = **23.1%**
  - n=20 → 有效阈值 = 40% × sqrt(20/30) = 40% × 0.816 = **32.7%**
  - n=30 → 有效阈值 = 40% × 1.0 = **40%**（恢复原始值）
- **或者**: 仅当 `n >= c1_min_trades` 时才执行 C4，否则跳过 C4 检测
- **推荐**: 采用自适应修正方案，保持C4的检测功能同时避免n不足时的误杀

### 4.2 校准建议

**空窗口处理是核心问题**: 当 n < 4（窗口数）时，必然有空窗口，空窗口的neutral判定会降低一致性比例，但它们既不是"稳定"也不是"不稳定"的证据。

| 维度 | 当前值 | 建议值 | 理由 |
|:----|:-----:|:-----:|:-----|
| direction_threshold | 0.01(1%) | **0.02(2%)** | 网格策略单笔收益0.05%，1%阈值过严。提升至2%减少噪声标记 |
| n_windows | 4 | **min(n_windows, n_trades)** | 窗口数不应超过记录数 |
| 空窗口处理 | neutral | **skip（不纳入统计）** | 空窗口不贡献方向，只在总记录中注明 |
| n < 4时 | 强制4窗口 | **min(4, max(2, n)) 窗口** | 动态调整窗口数以匹配数据 |

**增强方案**: 引入"数据充足性"前置检查:
```python
def temporal_validate(trades, min_trades_for_validation=8):
    if len(trades) < min_trades_for_validation:
        return "INSUFFICIENT_DATA — 数据不足，无法进行时间稳定性检验"
    # 否则正常执行
```

## 六、误杀专题分析

### 6.1 C4 (40% threshold) 误杀分析

**误杀判定标准**: C4 FAIL 的真正原因是样本量不足（n < 10），而非收益过度集中。

| 影响范围 | 影响报告 | 误杀比例 |
|:---------|:--------|:--------:|
| n=2的报告 | P1, P5, P6-84, P8 | **4/4 = 100%** |
| n=4的报告 | P6-3年 | **1/1 = 100%** |
| n≥30的报告 | P2(7回撤), P7(1540天) | **0/2 = 0%** |

**结论**: C4对 n ≥ 30 的报告无误杀。所有C4失败案例的根因是**C1样本不足**。建议通过自适应阈值（§5.1）或"C4仅在C1通过后执行"来消除误杀。

**推荐方案**: 自适应修正（见§5.1）。该方案在n=2时会将有效阈值降至10.3%，使得这些案例继续FAIL（正确——2笔交易确实不该通过验证）；在n=10时有效阈值23.1%，使得部分集中度适中的案例能通过；在n=30时恢复原始40%。

### 6.2 C5 (12/year) 偏严分析

**判定**: **偏严**。对网格/均值回归类策略构成系统性歧视。

| 策略类型 | 固有年交易数 | C5当前判定 | 建议阈值 |
|:--------|:-----------:|:---------:|:--------:|
| 网格 (n5_fixed) | 2~4/年 | ❌ FAIL | **2/年** |
| 网格 (n10_fixed) | 4~8/年 | ❌ FAIL | **2/年** |
| 趋势跟踪 | 6~15/年 | ✅/❌ 临界 | **6/年** |
| 日内 | 200+/年 | ✅ PASS | 保持12/年 |
| 因子IC | 250+/年 | ✅ PASS | 保持12/年 |

**推荐方案**: 策略类型自适应（见§5.1）。
