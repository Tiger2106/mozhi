## 1. 数据完整性

### 1.1 校验标准

- 回测期内所有交易日的数据完整覆盖，无缺失区间
- 各数据源在回测期内的覆盖率不低于 99.5%
- 加载数据量与实际数据量一致
- 回测报告中的数据库记录行数与数据库源表行数一致

## 2. 数据一致性

### 2.1 校验标准

- **数据源统一**: 所有输入来自同一数据仓库的同一版本
- **时间对齐**: 不同频率数据已正确对齐时间戳，无偏移
- **复权一致性**: 前后复权数据在全量范围匹配
- **异常值规则**: 已应用统一的异常值清洗规则（如N-sigma截断、Winsorize）
- **缺失值规则**: 所有缺失值处理方式已记录，且策略不依赖填充数据产生信号
- **数据无前视偏差**: 回测使用的数据在时间上不会"看到"未来信息

| # | 检查项 | 结果 | 检查方法 | 说明 / 问题描述 |
|:-:|:------:|:----:|:--------:|:--------------:|

| 2 | 不同频率数据时间戳对齐 | `<PASS/FAIL/NA>` | 时间轴交叉检查 | |
| 3 | 复权计算一致（前/后复权匹配） | `<PASS/FAIL/NA>` | 复权因子校验 | |
| 4 | 异常值处理规则已实施 | `<PASS/FAIL/NA>` | 对比原始与清洗后数据 | |
| 5 | 缺失值处理方法已记录 | `<PASS/FAIL/NA>` | 检查数据预处理日志 | |

## 4. 复现性

### 4.1 校验标准

- 回测代码版本已锁定（Git commit / tag），可重现
- 随机种子已锁定（若策略涉及随机性）
- 二次执行回测得到的结果偏差在可接受范围内（收益偏差 < 0.1%）
- 运行环境信息（OS/Python/依赖库版本）已完整记录
- **确定性策略说明**：若策略为完全确定性（无随机成分），则二次回测偏差 0.00% 是必然结果，无需作为问题标记

### 4.2 校验清单

| # | 检查项 | 结果 | 检查方法 | 说明 / 问题描述 |
|:-:|:------:|:----:|:--------:|:--------------:|
| 1 | 代码版本已锁定（commit/tag） | `<PASS/FAIL/NA>` | 确认Git记录（无Git管理标记NA） | |
| 2 | 随机种子已固定 | `<PASS/FAIL/NA>` | 确认seed设置 | |
| 3 | 二次回测结果偏差在阈值内 | `<PASS/FAIL/NA>` | 对比两次回测的绩效指标 | |
| 4 | **确定性策略→偏差0.00%说明**：若为确定性策略，二次回测偏差0.00%为必然结果，应在报告备注中标注"确定性策略，偏差0.00%为预期" | `<PASS/FAIL/NA>` | 确认报告中是否已标注说明 | 遗漏标注则标记为WARN |
| 5 | 运行环境已完整记录 | `<PASS/FAIL/NA>` | 检查环境配置文件 | |
