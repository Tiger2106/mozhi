# 数据方案设计：三方案对比

> **作者**: 墨衡 | **创建时间**: 2026-05-22T21:26

## 概述

从**效果出发**——即每个方案能支撑什么样的研究场景，反推标的数量、数据长度、频率需求。三个方案分别对应不同研究深度与工程投入的权衡。

---

## 方案A：最小可行 — 3标的×日频×3年

### 标的数量 + 选择策略
- **标的数**: 3只（当前已就绪的 601857/000001/600519）
- **覆盖行业**: 能源 / 银行 / 白酒，各一
- **选择逻辑**: 跨越三个非相关行业，代表不同经济周期的贝塔来源

### 数据长度
- **3年** (2023-2025)
- **为什么够**: 覆盖一轮完整小周期（反弹+震荡+下行）
- **缺点**: 仅覆盖一种宏观叙事周期，策略稳定性不可验证

### 数据频度
- **仅日频**（分钟级不需要——3只标的无法做分钟级任何有统计意义的研究）
- **工作量**: 零工程量（数据已采集就绪）

### 可达到的研究效果

| 研究类型 | 可行性 | 说明 |
|:---:|:---:|:---|
| IC/IR 因子检验 | ❌ 不可行 | 截面不足。3只标的×N个因子，相关系数无统计意义，P值不可靠 |
| 因子相关性矩阵 | ❌ 不可行 | 同上 |
| 分层回测（Quintile） | ❌ 不可行 | 需至少5只标的才能分一层 |
| 时间序列动量 | ✅ 可行 | 单标的时序信号（MA交叉、RSI）的回测 |
| 单因子单标的多空 | ⚠️ 有限 | 能做，但无统计可靠性 |
| 多窗口策略回测 | ✅ 基本可行 | 但只有3年，窗口划分有限 |
| Walk-forward 参数优化 | ❌ 不可行 | 3年数据不够切分训练/验证/测试集 |
| 统计套利/配对交易 | ❌ 不可行 | 配对交易需要同行业内标的相关性分析 |
| 板块轮动 | ❌ 不可行 | 每行业只有1只，无板块概念 |

### 工作量评估
- **数据采集**: 0（已就绪）
- **因子回填**: 单因子约2小时×3只=6小时/因子
- **其他**: 无

### 存储需求评估
- **日频数据存储量**: 3只×3年×4KB/年 ≈ 36KB（当前已有数据量，零增量）
- **因子仓库数据量**: ~30个基础因子×3只×750日 ≈ 108KB
- **运行时内存需求**: <50MB（日频OHLCV+因子加载，内存消耗极低）
- **磁盘总量**: ~0.2MB（可忽略不计，当前开发环境完全在内存承载范围内）

### 约束/风险
- **统计意义为零**: 任何基于截面比较的结论都不可靠
- **单数据源依赖**: Tushare Pro 单点故障
- **代表性问题**: 3只标的无法代表A股市场
- **研究价值边界**: 只适合个人策略的初步筛选，不适合正式因子研究

## 方案B：平衡方案 — 12只×日频+分钟×5年

### 标的数量 + 选择策略
- **标的数**: 12只
- **行业覆盖**: 每个A50核心行业选1-2只
  - 银行（招商银行 600036）
  - 保险（中国平安 601318）
  - 白酒（贵州茅台 600519）
  - 能源（中国石油 601857）
  - 新能源（宁德时代 300750）
  - 半导体（中芯国际 688981）
  - 医药（恒瑞医药 600276）
  - 消费（伊利股份 600887）
  - 基建（中国建筑 601668）
  - 证券（中信证券 600030）
  - 家电（美的集团 000333）
  - 科技（海康威视 002415）
- **选择逻辑**: 10-12个核心行业×各1-2只，覆盖A股主要驱动力

### 数据长度
- **5年** (2021-2025)
- **市场环境覆盖**: 
  - 2021: 结构性牛市（新能源/半导体）
  - 2022: 全面熊市（-20%）
  - 2023-2024: 震荡修复
  - 2025: 市场分化
  - **完整覆盖牛→熊→震荡三阶段**

### 数据频度
- **日频**: 全部12只——核心因子计算
- **分钟级**: 选3-4只高流动性标的（茅台/宁德/平安/招行）
  - 用于：日内信号回测、冲击成本建模、因子衰减分析
  - **分钟级生产化**: 需要从原型→定时任务+数据清洗+存储
    - MinuteCollector 原型改造 + 定时调度 (约3天)
    - 数据清洗/对齐/存储 (约2天)
    - 单标的分钟级磁盘空间: ~50MB/年/标的
    - 4只×5年 ≈ 1GB

### 可达到的研究效果

| 研究类型 | 可行性 | 说明 |
|:---:|:---:|:---|
| IC/IR 因子检验 | ✅ 可行 | 12只截面×5年约60个观测点，IC可计算，但IR的稳定性受限于低截面数 |
| 因子相关性矩阵 | ✅ 可行 | 12×12 相关性矩阵，消除多重共线性 |
| 分层回测 | ⚠️ 有限 | 勉强分3层（4只/层），分辨力不足 |
| 时间序列因子 | ✅ 可行 | 动量/反转/波动率因子在各标的上的表现 |
| 多窗口策略回测 | ✅ 良好 | 5年可划分3×1.5年窗口，跨时期稳定性验证 |
| Walk-forward 优化 | ⚠️ 有限 | 勉强可做，但样本外只有1-2年，可靠性一般 |
| 日内信号测试 | ✅ 可行（4只） | 分钟级可测试开盘/尾盘/事件驱动信号 |
| 执行成本建模 | ✅ 可行（4只） | 冲击成本/滑点模拟 |
| 跨行业联动效应 | ⚠️ 初步 | 12只×12行业，可以做简单的板块相关性 |
| 统计套利/配对 | ⚠️ 有限 | 同行业只有1-2只，配对选择受限 |

### 工作量评估
- **数据采集**:
  - 日频扩展（3→12只）: Tushare Pro 批量采集，约1天
  - 分钟级生产化: MinuteCollector 改造+部署，约5天
  - 数据对齐/校验: 约2天
  - **合计: ~8天**
- **因子回填**: 标准因子库 50个因子 × 12只 × 5年 ≈ 3000行，约3-5天
- **其他**: 分钟数据清洗/缺失值处理，约1天

### 存储需求评估
- **日频存储**: 12只×5年×4KB/年 ≈ 240KB
- **分钟级存储**: 4只×5年×50MB/年 ≈ 1,000MB（~1GB）
- **因子仓库**: 50因子×12只×1250日 ≈ 1.2MB
- **运行时内存**: 日频回测(~50MB) + 分钟级回测峰值(~500MB, 分钟数据批次加载) ≈ 550MB
- **磁盘总量**: ~1GB（分钟级是存储大头，日频+因子可忽略）

### 约束/风险
- **截面数量仍偏小**: 12只不够做Qunitile分层、多因子模型检验
- **分钟级只覆盖4只**: 结论不可外推到全市场
- **Tushare 依赖**: 高调用频次可能触发限流
- **偏差风险**: 选的都是大市值蓝筹，结论对中小盘不成立
- **分钟级数据质量**: AKShare 分钟数据可能存在缺失/异常值

## 方案C：完整方案 — 40只×双频×6年

### 标的数量 + 选择策略
- **标的数**: 40只
- **行业覆盖**: A50中选40只（剔除流动性差的5只 + 已退市/ST风险5只）
- **行业分布**:
  - 银行 ×4 (招行/工行/建行/农行)
  - 非银 ×3 (平安/中信/东方财富)
  - 消费 ×4 (茅台/五粮液/伊利/海天)
  - 医疗 ×4 (恒瑞/迈瑞/药明/片仔癀)
  - 科技 ×5 (海康/立讯/中芯/韦尔/北方华创)
  - 新能源 ×4 (宁德/比亚迪/隆基/通威)
  - 能源/材料 ×4 (中石油/万华/紫金/海螺)
  - 金融地产 ×3 (万科/保利/中信建投)
  - 基建/交通 ×3 (中建/中车/长江电力)
  - 家电/制造 ×3 (美的/海尔/格力)
  - 其他 ×3 (中移动/中免/金山办公)
- **选择逻辑**: 覆盖A50全部核心行业，每行业3-5只，保留同行业配对可能性

### 数据长度
- **6年** (2020-2025)
- **市场环境覆盖**:
  - 2020: 疫情后反弹 + 结构牛
  - 2021: 全面结构性牛市
  - 2022: 全面熊市
  - 2023: 弱复苏 → 磨底
  - 2024: 政策驱动反弹
  - 2025: 分化加剧
  - **六阶段全覆盖，walk-forward 窗口划分充裕**

### 数据频度
- **日频**: 全部40只（因子计算、策略回溯主线）
- **分钟级**: 全部40只
  - 分钟级生产化需要**全量改造**：
    - 生产级采集框架（多线程/断点续传/数据校验）
    - 分布式存储（分钟级×40×6年 ≈ 12GB）
    - 清洗流水线（跳变检测/异常值处理/复权对齐）
  - **必要性**: 高频交易研究 + 冲击成本 + 日内信号 + 收盘前信号

### 可达到的研究效果

| 研究类型 | 可行性 | 说明 |
|:---:|:---:|:---:|
| IC/IR 因子检验 | ✅✅ 强大 | 40只截面×6年≈240+观测点，IC/IR 统计显著 |
| 因子相关性矩阵 | ✅✅ 强大 | 40×40 矩阵，PCA/NMF 降维可用 |
| 分层回测（Quintile/Decile） | ✅✅ 可行 | 40只可做5分位（8只/层），统计意义充分 |
| 多因子模型构建 | ✅✅ 可行 | Barra-style 风格因子模型，Alpha 剥离 |
| 多窗口策略回测 | ✅✅ 强大 | 可切分4-5个独立窗口 |
| Walk-forward 优化 | ✅✅ 可行 | 24M训练+12M验证+12M测试×3轮 |
| 统计套利/配对交易 | ✅✅ 可行 | 同行业2-4只，协整检验有统计意义 |
| 板块轮动策略 | ✅✅ 可行 | 10个行业×3-5只，行业动量/轮动信号 |
| 日内信号 + 执行 | ✅✅ 可行（全量） | 分钟级全量覆盖，开盘/尾盘/事件信号 |
| 机器学习因子 | ✅✅ 可行 | 40只×150+因子×6年 ≈ 36万行，ML模型过拟合可控 |
| 风险归因分析 | ✅✅ 可行 | 行业/因子/单标的分层归因 |
| 组合优化/有效前沿 | ✅✅ 可行 | 40只可做MVO/BL等组合优化 |

### 工作量评估
- **数据采集**:
  - 日频批量采集（3→40只）: Tushare Pro，约3天
  - 分钟级全量生产化:
    - 框架重写（多线程/调度/监控）: 约5天
    - 全部40只分钟级采集+存储: 约5天
    - 数据清洗/对齐/校验: 约3天
  - **合计: ~16天**
- **因子回填**: 
  - 标准因子库 80个因子 × 40只 × 6年 ≈ 19,200行
  - 包含基本面/技术面/另类数据因子
  - 约7-10天（含计算验证）
- **其他数据工程**:
  - 分钟级存储方案设计（按标的+日期分片）: 约2天
  - 数据质量监控流水线: 约2天
  - 复权数据统一处理: 约1天
  - **合计: ~5天**

> **总工作量估算**: 约28-31天（约6周集中冲刺）

### 存储需求评估
- **日频存储**: 40只×6年×4KB/年 ≈ 960KB（~1MB）
- **分钟级存储**: 40只×6年×50MB/年 ≈ 12,000MB（~12GB）— 存储大头，需要合理分片策略
- **因子仓库**: 80因子×40只×1500日 ≈ 7.5MB
- **运行时内存**: 日频全量回测(~200MB) + 分钟级分批处理峰值(~1.5GB, 按标的分批次加载以避免内存溢出) ≈ 1.7GB
- **磁盘总量**: ~12GB（分钟级占绝对主导，建议按标的+日期分片存储以优化I/O）

### 约束/风险
- **工程量最大**: 约1.5个人月，资源需求高
- **分钟级质量风险**: AKShare 分钟数据可靠性需持续验证
- **存储压力**: 分钟级数据约12GB，加上因子仓库约200MB，当前本地存储可承受
- **Tushare Pro 调用量**: 40只×6年，日频调用量在API限制内，但需要合理规划批次窗口
- **计算开销**: 80因子×40只×6年 ≈ 19,200条因子值，单机Python可承受
- **数据维护成本**: 需要定期更新机制，而非一次性采集
- **过拟合风险**: 数据越多≠策略越好，需要严格控制回测方法论

## 建议路线

### 如果研究目标是：
1. **快速验证策略思路** → 方案A，但结论仅限参考
2. **正式因子研究** → **方案B是性价比最优选择**，12只×5年日频已可支撑IC/IR/因子相关性等核心研究
3. **生产级多因子系统** → 方案C是必要条件，40只才能构建有统计意义的多因子+组合优化

### 两步走建议
**Phase 1（2周）**: 落实方案B的日频部分
- 12只×5年日频采集就绪
- 核心因子库回填（50因子）
- 跑通 IC/IR 流水线 + 多窗口回测框架
- 确定研究框架和因子集合

**Phase 2（Phase 1 后+4周）**: 
- 方案B分钟级部分扩展（4只高流动性）
- 同步评估方案C必要性的数据验证结果
- 如果Phase 1因子IC稳定性良好 → 启动方案C全量扩展
