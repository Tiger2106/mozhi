## 1. 架构合理性（3.5/5 | WARN）

### 1.1 因子分析流水线设计

**评分: PASS**

核心流水线为 `run_exp003_q1.py` 驱动的独立窗口分析流程：

```
数据加载 → 状态分类(vol quantile) → 因子计算 → forward_returns缓存 → 
Bootstrap IC检验 → FDR BH校正 → L3稳定性验证 → 衰减率计算 → verdict判定 → 结果存储
```

**优点：**
1. **清晰的阶段分离**：数据加载 → 因子计算 → 统计检验 → 稳定性验证 → 结论判定，五个阶段职责清晰
2. **独立的配置文件**：q1_results.json中config段与design.md参数完全一致，参数集中管理
3. **version_tag嵌入**：每个输出文件携带commit hash，支持可追溯

**建议改进项：**
1. **缺少中间产物版本锁定**：decay_analysis.json 和 q1_results.json 虽然各自记录version_tag，但没有在文件名中嵌入commit标识。若以后修改代码后重新运行，产物难以区分版本。建议参考Mozilla平台约定：`q1_results_{commit_short}.json`
2. **缺少数据时间戳快照**：market_data.db在回测期间可能被后续任务修改。虽记录"数据截至2026-04-30"，但未做数据时间戳快照。建议在回测开始时记录 `data_snapshot_timestamp`

- **L3 pass门槛过低**：4维通过3维即标记L3_passed=true。这意味着：

## 2. 设计质量（3.5/5 | WARN）

### 2.1 Bootstrap检验实现

**算法概要（反向工程自 exp_bootstrap.py）：**
1. 排列因子值（shuffle）1000次，生成零分布
2. 计算经验p值 = (count(null_IC ≥ observed_IC) + 1) / (n_shuffles + 1)
3. 排列的是因子值（非IC序列）→ 前视偏差防护设计

**实现评估：**

| 设计维度 | 评分 | 说明 |
|:---------|:----:|:------|
| 排列对象 | ✅ 正确 | 排列因子值而非IC序列，与design.md一致，合理 |
| 迭代次数(1000) | ✅ 合理 | 对于α=0.05，1000次提供±1.4%的p值精度，足够 |
| 伪随机数处理 | ❌ WARN | **随机种子未锁定**，QA报告已标记。bootstrap过程使用numpy.random.shuffle，与numpy默认可变种子绑定。虽不改变统计显著性结论，但影响精确复现 |
| 极端值处理 | ✅ 合理 | bootstrap为无分布方法，天然对异常值鲁棒 |

**WARN-01：随机种子锁定缺失**
- 影响：精确复现需在相同硬件/OS/pandas版本下运行多次，IC值的第15位有效数字可能略有差异
- 修复成本：低。在bootstrap函数入口加 `np.random.seed(asset_parameter)` 即可
- 建议：将因子值hash值作为seed的一部分，保证：(1) 相同输入→相同输出，(2) 不同输入→不同随机序列（避免全局种子重复使用导致的p值偏差）

### 2.2 forward_returns缓存设计

**设计目标**：避免在每次bootstrap迭代中重复计算forward_returns，提升性能约2x。

**实现检查（来自P2自一致性验证）**：
- P2验证中优化前/后IC_diff=0.0，p_diff=0.0 → 缓存逻辑不影响结果正确性
- 加速比1.1x~2.08x → 有效

**设计质量评分：PASS**

**设计细节建议：**
- 缓存命中检查应该在factor+period+state三重key上做精确匹配，从decay_analysis.json的数据结构看，确实做了（3层嵌套字典），设计正确
- 缓存生命周期应限定在一次q1_results.json生成周期内，而非跨session持久化。从报告看，缓存是内存级，符合预期

### 2.3 状态分类设计（high/mid/low vol）

**分类方法**：基于训练期波动率3分位数，在全样本上应用固定阈值。

**优点：**
- 分位数分类避免了绝对阈值对宏观波动水平的依赖
- 固定阈值确保了训练期/验证期分类标准一致

**设计问题：**

**WARN-02：状态标签的通胀问题**
- 训练期高波阈值（上1/3分位数）套用到验证期后，受2020~2022高波动市场影响，验证期"高波"可能对应完全不同的波动率绝对值
- 结果：验证期high_vol样本仅1.3K（训练期15K的1/10），说明大部分样本"降级"成mid/low
- 影响：高波状态的统计检验力严重不足，这个设计决策使得"高波状态跨窗口衰减"这个核心问题的结论可靠性打折

**推荐修复：**
- 在报告中增加"状态分类稳定性分析"：展示训练期vs验证期的波动率阈值分布对比，以及各状态样本比例的漂移量
- 考虑增加"相对波动率"或"滚动波动率百分位数"的替代分类，验证结论的稳定性

### 2.4 衰减率计算方法

**公式**：`decay_rate = (val_IC - train_IC) / |train_IC|`

**评价：**
- 分母取绝对值修正了符号方向问题：当train_IC为负且val_IC向零靠近时，衰减率正偏，逻辑正确
- 阈值判定：≤50%为通过 → 合理选择

**WARN-03：负值放大效应**
- 当train_IC≈0时（如l_vol_rsi_std/low_vol/10d: train_IC=+0.0025），衰减率计算敏感度过高。任何微小的IC变动都产生巨大的"衰减率"（88.8%）
- 虽然这个组合的verdict实际是FAIL（方向翻转），但若方向一致且train_IC≈0时，衰减率不应作为主要判定指标
- **建议**：增加条件——若|train_IC| < 0.01（IC绝对值低于1%），衰减率不计入主判断，只作参考

### 3.1 是否过度设计？

| 设计元素 | 必要性 | 评价 |

| L3 4维稳定性 | 适度 | 4维提供了时间+截面+跨窗口三重视角 |

| P2自一致性验证 | 必要 | 优化必须保证计算等价性 |
| **综合** | **适度** | 没有冗余或花哨的设计元素 |

**没有发现过度设计。** 每个统计检验组件都有明确且必要的功能目的。

### 3.2 是否有缺失的边界条件？

**缺失项（2个）：**

1. **验证期样本不足时的fallback逻辑**：当high_vol样本从15K降至1.3K时，统计检验力的下降幅度没有被量化。设计上没有"样本量不足则降级结论"的条件分支
   - 建议：增加 `n_samples >= 5000` 的样本量门槛，低于此值时verdict自动降一级（PASS→WARN, WARN→FAIL）

2. **IC为0附近的判定边界**：衰减率计算在train_IC≈0时的数学敏感性问题（§2.4 WARN-03），缺乏额外的保护逻辑

### 3.3 设计契约检查

**正向（正确的设计决策）：**
- Spearman Rank IC而非Pearson：A股因子与收益关系通常非线性和非正态，秩相关更鲁棒
- BH法而非Bonferroni：18个检验的严格控制自由度损失太大
- 排列因子值而非IC序列：因子的前视偏差防护的最安全方式
- 独立窗口而非滚动窗口：避免了数据污染风险，是跨窗口验证的严格版本

**负向（需关注的设计边缘）：**
- 训练期IC计算基于整个训练期整体，而非滚动估计。这意味着训练期内部的IC非平稳性被平均化了（如2008极端市场的因子表现与2017相差多大的信息没有保留）
- Verify方法：可以在报告中补充"训练期时间分割IC稳定性"数据

**评分：PASS**。因子衰减处理是Q1分析的核心目的，实现完整且正确。

### 4.2 方向翻转处理

**实现覆盖：** 方向翻转通过 `direction_consistent` 标志约束，翻转即→FAIL。正确处理 ✅

**设计扩展建议：**
- 方向翻转没有区分"统计显著的方向翻转"vs"不显著的方向翻转"。如 `l_vol_rsi_std/high_vol/10d` 是显著翻转（train=+0.051显著 → val=-0.067显著），而 `l_vol_rsi_std/low_vol/10d` 是不显著翻转（train=+0.0025不显著 → val=-0.0003不显著）。前者的信息价值远高于后者，但verdict都是FAIL
- **建议**：在verdict判定中增加"significance-aware"翻转检测——只有双显著的方向翻转才标记"强力失败"

### 4.3 样本不平衡处理

**评估：WARN ⚠️ — 未充分处理**

| 状态 | 训练期样本 | 验证期样本 | 比例变化 |
|:----:|:----------:|:----------:|:--------:|
| low_vol | ~97K | ~50K | -48% ✅ 可控 |
| mid_vol | ~33K | ~6K | -82% ⚠️ |
| high_vol | ~15K | ~1.3K | **-91% 🔴** |

**问题：**
- 验证期high_vol样本从15K降至1.3K（约91%衰减），n=1,312的bootstrap检验统计检验力极低
- mid_vol同样从33K降至6K（-82%），部分检验（5d n=6,074, 10d n=6,062, 20d n=6,038）勉强可用但边缘
- 当前设计没有针对样本量不足的统计检验力分析或降级处理

**建议修复：**
1. 增加 `min_samples=5000` 的有效检验下限
2. 在报告中增加"各状态样本量对比表"（验证期/训练期比例、衰减量）
3. 对high_vol验证期结果标注**"检验力受限，结论置信度降低"**的警告

### 4.4 市场环境变更处理

**评分：WARN**

- 2020~2026期间A股经历了剧烈的市场结构性变化
- 已在校验中捕捉到：TrendQuality high_vol的IC从+0.011(不显著)→+0.183(显著)，极可能是市场微观结构变化驱动的"
- 当前设计没有对"市场状态本质变化"的检测机制

**建议：**
- 增加**状态转移矩阵**分析：训练期各状态中样本的波动率范围 vs 验证期各状态中样本的波动率范围
- 增加**时段分割IC分析**：验证期5个子时段（2020/2021/2022/2023/2024~2026）的IC时序
