## §1 Layer Q 评分体系详解

### 1.1 什么是 Layer Q

Layer Q 是 Mozhi 系统的**可信度审计层**，位于回测管线的最后一环。它的任务不是看你的策略赚了多少钱，而是判断这个钱**赚得是否可信**。

通俗地说：一个骗子也可以在纸上画出漂亮的净值曲线。Layer Q 的工作就是拆穿他。

### 1.2 A~F 六级评分

Layer Q 将策略的可信度分为六个等级。**注意：评级越高不等于收益越高，只代表结论越可信。**

| 等级 | 复合 R 值 | 含义 | 对本金的态度 |
|------|-----------|------|-------------|
| **A** | 0.80 ~ 1.00 | 高度可信：全维度验证通过，无瓶颈 | ✅ 可上核心仓位 |
| **B** | 0.65 ~ 0.79 | 中高可信：大部分维度通过，存在轻微不足 | ✅ 可上，仓位管理 |
| **C** | 0.50 ~ 0.64 | 中可信度：存在明显过拟合风险 | ⚠️ 谨慎，辅助判断 |
| **D** | 0.30 ~ 0.49 | 低可信度：多个维度不通过，需要复审 | ❌ 不建议上线 |
| **E** | — | 不可信：明显过拟合 + 参数尖峰 | ❌ 禁止上线 |
| **F** | 0.00 ~ 0.29 | 完全不可信：严重统计或逻辑缺陷 | ❌ 删除研究 |

**特别说明：**
- 指南正文中写的是 A~F 六级，但当前代码实现中仅实现 A/B/C/D/F 五级（E 级待补充）。你看到的报告中若显示 E，说明该发现年久失修——建议直接标记为不可用。
- 评级计算的核心理念是**短板效应**：复合 R 值采用加权调和平均（不是简单的平均数）。这意味着——如果一个维度评分极低（如参数稳健性只有 0.1），即使其他五个维度都是满分，整体评级仍然会被严重拖低。这符合"木桶原理"：一个策略的可靠程度取决于它最弱的一环。

### 1.3 六维评分框架

Layer Q 的可信度评分来自六个独立维度。每个维度的得分在 0.0 ~ 1.0 之间：

| 维度 | 权重 | 验证什么 | 通俗解释 |
|------|------|----------|---------|
| **Q1 存在性** | 20% | 是否存在统计意义的规律 | 这真的是个策略，还是纯噪声？ |
| **Q2 参数稳健性** | 20% | 参数变化时收益是否稳定 | 换几个参数会不会崩？ |
| **Q3 市场状态适配** | 15% | 不同 Regime 下的一致性 | 是不是只在牛市有用？ |
| **Q4 资金容量** | 15% | 能承载多少资金 | 大资金进来还能跑吗？ |
| **Q5 时间稳定性** | 15% | 随时间推移是否衰减 | 明年还管用吗？ |
| **Q6 样本外存活** | 15% | 在未见过的数据上是否有效 | 拿没见过的数据验一遍？ |

**各维度低于 0.65 会标记为"瓶颈维度"**，并在报告的综合评级中突出显示。

### 1.4 硬门禁机制（Hard Gate）

在某些情况下，不满足基本条件时，系统会绕过计算逻辑直接判 F 级。

**C1 硬门禁**：如果 Existence 检查发现"交易数量少于 30 笔"或"没有任何交易记录"，系统直接判定：复合 R ≤ 0.30，评级自动锁定为 F。

为什么这么严格？统计学上，样本量低于 30 时，任何置信区间估计的可靠性都非常低。你无法区分这是市场规律还是噪音。

### 1.5 评级 vs 收益——正交的

一个常见误区：**"A 级策略一定赚钱很多"——不对。**

正交通理：可信度和盈利能力是相互独立的。一个 A 级策略可能只赚很少的钱，但它告诉你"这个规律确实存在"。一个 C 级策略可能短期赚了很多钱，但那只是因为运气好，实盘中你大概率会亏回去。

Layer Q 的核心价值就在于此：**它帮你区分的不是好策略和坏策略，而是可信的策略和不可信的研究。**

## §2 WalkForward 验证方法详解

### 2.1 为什么要做 WalkForward

假设你发现一套参数让某个策略在过去三年的回测中看起来非常好。问题是：这是真的因为方法有效，还是因为你运气好选对了时间段？

传统方法只做一次全样本回测——用全部数据来"验证"你自己选出的参数。这本质上是在做**开卷考试**：你知道答案再去翻书。

WalkForward 模拟了**闭卷考试**的过程：

```
时间轴 →
|─── 训练期 ───|─── 测试期 ───|
  在这个窗口选参数    拿选好的参数在这里验证
                 → 向前滚动 →
           |─── 训练期 ───|─── 测试期 ───|
             重新选参数      在新窗口验证
```

每次只让模型在训练期"学习"参数，然后在训练期之后的测试期"考试"。通过多次这样的滚动验证，你可以判断：**选出的参数是真的有效，还是只是刚好适合某段历史数据。**

### 2.2 怎么做（方案 C — 默认方案）

系统默认使用 **WalkForward 方案 C**，包含 5 个滚动窗格。每个窗格的结构如下：

| 窗格 | 训练期 | 测试期 | 说明 |
|------|--------|--------|------|
| W1 | 2023-01-01 → 2023-09-09 | 2023-09-11 → 2023-12-03 | 最早窗口 |
| W2 | 2023-08-11 → 2024-04-19 | 2024-04-20 → 2024-07-12 | 滚动步长约 7 个月 |
| W3 | 2024-03-20 → 2024-11-27 | 2024-11-28 → 2025-02-19 | 滚动步长约 7 个月 |
| W4 | 2024-10-27 → 2025-07-06 | 2025-07-07 → 2025-09-28 | 滚动步长约 7 个月 |
| W5 | 2025-06-05 → 2026-02-11 | 2026-02-12 → 2026-05-14 | 最新窗口 |

**每个窗格做什么：**
1. 取训练期数据（约 252 个日历日，折合约 180 个交易日）
2. 在训练期内做参数扫描，找到最优参数组合
3. 用这个最优参数在测试期（约 84 个日历日，折合约 60 个交易日）跑一次回测
4. 记录本次测试的 Sharpe、年化收益、最大回撤等绩效指标
5. 向前滚动到下一个窗格，重复 1~4

**5 个窗格跑完后，你会得到 5 个独立的测试结果。** 如果 5 次测试的表现都大致稳定，说明策略的参数是可靠的。如果第 1 次和第 5 次的参数完全不同，性能相差巨大——说明策略与时间段高度相关，不可靠。

### 2.3 结果怎么看：四个核心指标

#### ① 平均 WFE（WalkForward Efficiency）

```
WFE = 测试期 Sharpe / 训练期 Sharpe
```

- WFE > 0.7：优秀，训练期发现的最优参数在测试期仍然有效
- WFE 0.5 ~ 0.7：尚可，有一定程度的参数衰减
- WFE < 0.5：不理想，参数过拟合倾向明显
- WFE 为负：测试期出现亏损，策略在未见过的数据中完全失效

**举例：** 训练期 Sharpe = 1.8，测试期 Sharpe = 0.9，WFE = 0.9 / 1.8 = 0.50。这说明参数虽然在训练期表现很好，但在实际"考试"中损失了一半的效能——这种情况需要警惕。

#### ② WFE 标准差

衡量 5 个窗格之间 WFE 的波动程度。如果标准差很大（> 0.3），说明策略的表现极度依赖选择哪个时间段做训练——这是一个危险信号。

#### ③ 参数重用率

```
参数重用率 = 最常选参数出现次数 / 有效窗格总数
```

- 重用率 > 60%：好，最优参数在不同时间窗口保持一致
- 重用率 30% ~ 60%：中等，参数有一定漂移
- 重用率 < 30%：差，每个窗口选出来的最优参数都不一样

**真实案例：** 对某网格参数空间扫描发现，5 个窗格选中了 5 种完全不同的参数组合。参数重用率 = 20%。这意味着"最优参数"的概念本身就是不稳定的——不同时间窗口下"最好"的参数迥然不同，说明策略不具备跨时间的一致性。

#### ④ 非交易窗格数

有些窗格的测试期可能没有任何交易。如果超过 1~2 个窗格出现这种情况，说明策略的信号密度过低——大多数时间你在空仓等待，收益的有效时间覆盖不足。

### 2.4 警示信号

| 现象 | 含义 |
|------|------|
| 第 1 个窗格收益很高，后面逐步递减 | 策略在退化——可能规律正在消失 |
| 5 个窗格的 WFE 出现明显正负交替 | 策略时而有效时而无效——不稳定 |
| 某个窗格的测试期出现极端亏损 | 参数在该时间段完全失效——需要检查该时段的市场状态 |
| 最优参数类型在每个窗格都切换（如 W1 用 arithmetic、W2 用 geometric） | 参数空间本身不稳定——不是一个可靠的策略 |

## §3 参数岛 vs 参数尖峰：判别方法与案例

### 3.1 两个概念

当你对一个策略做参数扫描（即遍历所有可能的参数组合，看每个组合下的收益），结果会形成一个"参数空间收益地图"。

**参数岛（Parameter Island）**——收益地图上出现**一片连续的高原**：
```
收益
  ^      ~~~~~~
  |     ~~~~~~~~
  |    ~~~~~~~~~~
  |   ~~~~~~~~~~~~
  +-----------------> 参数值
```
特点：参数在小范围内变动，收益只有微小波动。这说明策略抓住了**真实**的市场规律——规律是稳定的，不会因为参数微调就失效。

**参数尖峰（Parameter Spike）**——收益地图上出现**一个孤立的峰**：
```
收益
  ^         *
  |        / \
  |       /   \
  |      /     \
  +-----------------> 参数值
```
特点：只有一个特定的参数组合收益极高，稍微偏离一点点，收益就垂直下降。这说明策略几乎肯定是在**拟合历史噪音**——它"记住"了某段特定数据中的偶然模式，而不是发现了真实的规律。

### 3.2 判断方法

#### 方法一：热力图观察法

在回测报告中，参数扫描结果以热力图呈现。观察热力图的色彩分布：

- **绿色/暖色连成一片** → 参数岛 ✓
- **只有孤立的红点，周围全是蓝色** → 参数尖峰 ✗

具体怎么看：
1. 找到报告中的"收益 vs 参数"热力图（通常在参数扫描章节）
2. 看高收益区域是否连续（即相邻的参数组合收益也高）
3. 如果高收益区域被低收益区域包围，且面积很小（只占图面积的不到 5%）→ 尖峰警报

#### 方法二：临界距离法

问一个问题：**参数微调 1 个单位（例如 MA(5,20) → MA(5,21)），收益下降了多少？**

- 下降 < 10%：参数岛（稳定）
- 下降 10% ~ 30%：边缘状态，需要更多数据确认
- 下降 > 30%：尖峰嫌疑。**极端情况：下降 50% 以上 → 确认过拟合**

#### 方法三：WalkForward 交叉验证法

参考 §2.3 的"参数重用率"。如果 WalkForward 测试中：
- 5 个窗格选出的最优参数**始终在一个小范围内波动**（例如 MA 周期始终在 4~6 之间）→ 有参数岛 ✔
- 每个窗格选出的最优参数**大幅跳跃**（例如 W1 选 5/20，W2 选 20/60，W3 选 3/10）→ 参数尖峰 ✗

### 3.3 案例：R=0.02 的技术解释

**案例背景：** 某用户在回测报告中看到 WalkForward 的 R 值显示为 0.02。

#### 先确认 R 值在系统中的含义

在 Mozhi 系统中，"R 值"是一个有多义性的术语，不同场景下指代不同指标。请务必区分：

| 上下文 | R 值的含义 | 正常范围 |
|--------|-----------|---------|
| 指南正文 §7.2 的 R 值 | 稳健度系数（R > 0.7 才可信） | 0 ~ 1 |
| 代码中 Q7 的 composite_r | 复合可信度评分 | 0 ~ 1 |
| WalkForward 的 WFE | WalkForward 效率比 | 通常 0 ~ 1，可超过 1 |

当说"R=0.02"指向的是复合可信度评分（composite_r）时，这意味着：

**composite_r = 0.02 ≈ 评级 F（极低置信度）**

这个数字的含义拆解如下：

**根据加权调和平均公式：**
```
R = Σ(wi) / Σ(wi / si)
```
其中 `wi` 是权重，`si` 是各维度得分。

能算出 0.02 这个极低的值，只有两种可能：

**情况 A：硬门禁触发**（最可能）
- Existence 检查发现交易数 < 30 笔 → 自动锁定 composite_r ≤ 0.30
- 如果同时存在其他维度评分极低（如 Existence 维度只有 0.02）→ R = 0.02
- **结论：样本量极度不足，策略根基不稳，完全不可信**

**情况 B：六维评分全部极低**
- 例如：Existence=0.10, Robustness=0.05, Regime=0.05, Capacity=0.30, Temporal=0.05, OOS=0.02
- 加权调和平均的计算：(0.20+0.20+0.15+0.15+0.15+0.15) / (0.20/0.10 + 0.20/0.05 + 0.15/0.05 + 0.15/0.30 + 0.15/0.05 + 0.15/0.02)
- = 1.0 / (2.0 + 4.0 + 3.0 + 0.5 + 3.0 + 7.5) = 1.0 / 20.0 = 0.05
- 再加上调和平均天然对低分敏感（比算术平均低很多）→ R ≈ 0.02

**当你在报告中看到 R=0.02，你的行动指南：**
1. **第一反应：放弃这个研究结论。** 0.02 意味着系统认为"几乎没有任何可信度"。
2. **第二件事：打开 Existence 检查章节**，看 C1（最小交易数）是否通过。如果 C1 不通过（交易数 < 30），这就是失败的根本原因。
3. **第三件事**：如果 C1 通过了（交易数 >= 30），再看哪个维度得分最低。最低的 1~3 个维度就是评级被拉低的"瓶颈"。
4. **决定：** 如果是样本量问题 → 扩大研究窗口，增加数据量。如果是参数过拟合 → 换策略方法。如果是时间稳定性问题 → 缩短参数有效期。但无论如何，R=0.02 的参数组合**在任何情况下都不应被用于实盘或资金配置。**

## §4 ExistenceValidator/ExistenceCheck 的工作原理

### 4.1 它回答什么问题

ExistenceValidator 回答的是策略研究最基本的一个问题：**这个策略真的存在吗？**

听起来很奇怪——明明跑出了回测结果，"存在"还需要证明？是的，因为很多"策略"只是数据噪音的产物。

**ExistenceValidator 的核心任务：排除纯噪声。**

### 4.2 六项检查

ExistenceValidator 执行的六项检查，每一项都在问一个具体问题：

#### C1 最小交易数（权重 30%）

**检查：** 交易次数 ≥ N（默认 30 次）

**为什么重要：** 30 是统计学上的最小样本量门槛。如果连 30 次交易都没有，任何结论都是小样本噪声。

**例子：** 某策略在 3 年时间里只产生了 12 次交易信号。12 < 30 → C1 不通过。系统直接判定：样本量不足，不能下任何结论。

**在代码中是硬门禁（Hard Gate）——C1 不通过时，整个策略评级直接锁定为 ≤ 0.30（F 级）。**

#### C2 多 Regime 覆盖（权重 15%）

**检查：** 交易发生至少覆盖 K 种不同的市场状态（默认 ≥ 2 种）

**为什么重要：** 如果所有交易都发生在 TREND_UP（牛市）中，你无法判断这个策略在熊市或震荡市中的表现。

**例子：** 某 MA Cross 策略的交易全部分布在 TREND_UP 市场状态中。C2 不通过 → 策略只在牛市有效，进入震荡市必亏。

#### C3 多年度覆盖（权重 15%）

**检查：** 交易的日期跨度 ≥ T 年（默认 ≥ 2 年）

**为什么重要：** 跨年度的数据可以检验策略是否经得起市场环境变化。一年以内的数据可能只覆盖了一种市场状态。

**例子：** 你在 2019 年测试了一个策略，2019 年是全年牛市。如果不跨年，你可能以为策略很厉害。但加上 2018 年（熊市）或 2020 年（疫情暴跌）的数据，结论可能完全不同。

#### C4 非单段收益（权重 10%）

**检查：** 单笔最大收益（绝对值）在总收益中的占比 < 40%

**为什么重要：** 有些策略的收益完全依赖于一笔"中彩票"式的交易。去掉这笔交易，策略就是亏损的。这是典型的过拟合信号。

**例子：** 总收益 100 万，其中一笔交易赚了 60 万。C4 不通过 → 60% 的收益来自一次交易，策略的成功高度依赖单次运气。

#### C5 信号密度（权重 15%）

**检查：** 年平均交易/信号次数 ≥ 默认 12 次/年

**为什么重要：** 如果信号太稀疏（一年只有 1~2 次交易），策略没有足够的实践基础。当然，极低频策略（如长期持有）可能是合理的，但系统会在报告中进行标注。

**例子：** 某策略年均信号数 = 8 次 < 12 → C5 不通过，但报告中会标注为"低频策略边界"——你可以手动判断是否接受。

#### C6 样本分布（权重 15%）

**检查：** 交易在时间轴上是否均匀分布（单一窗口占比 ≤ 50%）

**为什么重要：** 如果所有交易都集中在某一个小时间段（比如 2015 年 6 月的股灾期间），策略的有效性就被那段特殊时期的条件绑架了。

**检查方法：** 系统将整个时间跨度平分为 10 个等宽窗口，然后计算每个窗口内的交易数量占比。如果某个窗口的交易占比超过 50%，说明交易高度集中。

**例子：** 一个策略在 3 年跨度中有 80% 的交易集中在中间的 6 个月（窗口 4~5）。C6 不通过 → 策略在大多数时间里几乎没有信号，只在某一段特殊行情中有效。

### 4.3 综合判定

六项检查都通过 → `exists = True`，置信度按加权平均计算（C1 占 30%，其余各占 10-15%）。

任何一项不通过 → `exists = False`，但置信度并不会归零，而是按已通过项的比例计算。

**注意：即使 exists = False，系统也不建议你立刻放弃。** 看 fail_reasons 找到具体哪项不通过，如果是数据量不够（C1）或窗口太短（C3），可以考虑扩充数据后重新验证。

### 4.4 在实际报告中的位置

Existence 检查结果出现在回测报告的第一个验证章节。形式如下：

```
═════════════════════════════════════
[ExistenceValidator] 存在性验证
═════════════════════════════════════

✅ C1 最小交易数:     n=47 >= 30          PASS
✅ C2 多Regime覆盖:   n=3  >= 2           PASS  (TREND_UP/OSCILLATION/PANIC)
❌ C3 多年度覆盖:     T=1.3年 < 2.0年     FAIL  (只覆盖了1.3年)
✅ C4 非单段收益:     最大占比22.3% < 40%  PASS
❌ C5 信号密度:       n=36.2/年 >= 12      PASS
❌ C6 样本分布:       最大窗口占比 38.5% < 50% PASS

Result: EXISTS=False | Confidence=0.72 | 1项未通过
瓶颈维度: C3 多年度覆盖 (T=1.3年)
```

看到上面的报告，你的决策应该是：
- 总体置信度 0.72 其实不错
- 但 1.3 年的数据跨度不够 → **延长研究窗口到 2 年以上，重新跑一次**
- 如果延长期限后的结果依然正向，这个策略就是可信的

## §5 当前系统可信度覆盖现状

### 5.1 总体情况

Mozhi 从设计上规划了 6 个验证维度（Q1~Q6 + 门禁系统），但在当前版本中，**部分维度的实现尚在原型阶段**。下面是每个验证维度的状态：

| 验证维度 | 核心模块 | 当前状态 | 产出物 |
|---------|---------|---------|--------|
| **Q1 存在性** | `existence_validator.py` | ✅ **已就位** | C1~C6 六项检查 + ExistenceResult |
| **Q2 参数稳健性** | `q2_robustness_validator.py` | ✅ **已就位** | 参数扫描 → 参数岛检测 → R 值 |
| **Q3 市场状态适配** | `q3_regime_validator.py` | ✅ **已就位** | 各 Regime 分层绩效对比 |
| **Q4 资金容量** | `q4_capacity_validator.py` | ⚠️ **原型阶段** | 容量估算，但输出维度单一 |
| **Q5 时间稳定性** | `q5_temporal_validator.py` | ⚠️ **原型阶段** | 时间分片绩效分析 |
| **Q6 样本外存活** | `q6_oos_validator.py` | ⚠️ **原型阶段** | 样本外测试基本框架 |
| **Q7 评级聚合器** | `confidence_rating.py` | ✅ **已就位** | composite_r + A~F 评级 |
| **Q8 失效归因** | `q8_failure_attribution.py` | ⚠️ **原型阶段** | 自动归因推理 |
| **Gate 集成** | `gate_integration.py` | ✅ **已就位** | G1/G2/G3 自动写 Q_FAILURES |

### 5.2 各维度详细说明

#### Q1 存在性（已就位）

- ExistenceValidator 已完成：6 项检查全部实现
- 硬门禁（C1 最小交易数 < 30 → 自动判 F）已生效
- 输出格式完整：fail_reasons + details + confidence
- 集成路径：ExistenceValidator → ConfidenceAggregator → Q7 → RatingResult

**如何使用：** 每次回测结果中自动触发。用户无需手动调用。

#### Q2 参数稳健性（已就位）

- 通过参数扫描 + WalkForward 联合验证
- 参数岛检测基于 WFE 和参数重用率
- WalkForward 方案 C（5 窗格）已固定为默认方案
- 支持自定义聚焦参数空间（focused_params）来缩小扫描范围加速

**如何使用：** 在回测配置中需要指定 `run_walkforward=True` 来启用。

#### Q3 市场状态适配（已就位）

- 将回测结果按 Regime（TREND_UP / OSCILLATION / PANIC / LOW_VOL）进行分层
- 对比各 Regime 下的 Sharpe、收益、最大回撤
- 核心输出：跨 Regime 绩效对比表

**如何使用：** 自动触发，依赖 RegimeAnalyzer 的市场状态标签。

#### Q4 资金容量（原型阶段）

- 当前实现为基本容量估算器，通过模拟不同资金量级的成交偏差来计算容量上限
- **尚未实现：** 逐笔流动性建模、大单成交冲击的精细模拟
- **具体限制：** 不支持对不同资金曲线做离散化切割和统计显著性检验（P4 里程碑的实现依赖于 `q4_capacity_validator.py` 的下一步开发）

**已知问题：** 容量估算目前仅输出"上限金额"一个数字，没有给出容量置信区间——这在本质上是不够的。

#### Q5 时间稳定性（原型阶段）

- 通过将数据切割为多个时间分片，比较各分片绩效的一致性
- **尚未实现：** 参数的自动衰减检测（如通过时间序列回归判断参数漂移趋势）、稳定性置信区间

#### Q6 样本外存活（原型阶段）

- 提供基本的样本外测试框架：保留最后 20% 数据作为验证集
- **尚未实现：** 跨品种样本外测试、独立的对比基线
- **关键缺失：** 样本外测试的数据构建没有自动排除数据集泄露问题（如同一只股票的训练集和测试集有日期重叠）

#### Q7 评级聚合器（已就位）

- 六维加权调和平均计算 composite_r
- A/B/C/D/F 评级自动映射
- 瓶颈维度自动识别
- 已集成到 report_builder 生成 PDF

### 5.3 当前限制总结

| 能力 | 当前状态 | 预期全功能时间 | 影响 |
|------|---------|--------------|------|
| 参数岛检测 | ✅ 完善 | — | 核心功能可用 |
| WalkForward | ✅ 完善 | — | 核心功能可用 |
| Existence 验证 | ✅ 完善 | — | 核心功能可用 |
| Regime 分层 | ✅ 完善 | — | 核心功能可用 |
| 资金容量上限 | ⚠️ 基础版 | Phase 4 | 大资金策略不可靠 |
| 时间稳定性 | ⚠️ 基础版 | Phase 4 | 策略衰退不可追踪 |
| 样本外存活 | ⚠️ 基础版 | Phase 4 | 跨数据集验证受限 |
| 失效归因推理 | ⚠️ 基础版 | Phase 4 | 失败诊断需人工 |

**给研究者的建议：**
- 如果你的策略资金量很小（<100万）：Q1~Q3 的验证已经足够
- 如果你的策略资金量很大（>1000万）：Q4（资金容量）的验证不可跳过，但当前版本只能给近似值——与 TradeMaster 联合测试更稳妥
- 如果你想长期使用某策略：Q5（时间稳定性）需要你手动监控策略表现，定期与历史数据对比，因为自动衰减检测尚未实现
- 如果你想跨品种推广策略：Q6（样本外存活）需要你手动选择另一只相关标的来验证，因为跨品种测试框架尚未自动化

### 6.2 快速阅读法：三步走

如果时间有限，按以下顺序阅读：

**第一步：翻到第 1 章摘要 → 看最终评级**

```
═════════════════════════════════════
研究摘要
═════════════════════════════════════
方法:     MA Cross (5,20) — China Petroleum (601857)
窗口:     2023-01-01 → 2026-05-14
评级:     B (中高可信度)      ← 第一眼要看这个
复合 R:  0.72
瓶颈:    Q5 时间稳定性 (0.45)
═════════════════════════════════════
```

- 看到 A / B → 可以看下去
- 看到 C → 谨慎，特别关注瓶颈维度
- 看到 D / F → **不需要看收益数据，直接关闭报告**

**第二步：跳到第 9 章评级汇总 → 看瓶颈维度**

如果评级是 C 或 B 但不太高，关键问题在于瓶颈维度。找到评分最低的 1~3 个维度。

- 如果瓶颈是 Q2（参数稳健性）→ 去第 7 章看 WalkForward 的 WFE
- 如果瓶颈是 Q3（市场状态）→ 去第 8 章看 Regime 分层，看是哪种市场亏的
- 如果瓶颈是 Q1（存在性）→ 去第 6 章看 C1~C6，找具体哪项检查未通过

**第三步：翻到第 7 章 WalkForward → 看四个指标**

- 平均 WFE > 0.7？→ 好
- WFE 标准差 < 0.2？→ 好（各窗格稳定）
- 参数重用率 > 60%？→ 好（参数一致性）
- 非交易窗格数 = 0？→ 好（信号密度足够）

如果其中 2 项以下不达标，策略基本可信。如果 3~4 项不达标，即使第 1 步看到的评级是 B，也要怀疑。
