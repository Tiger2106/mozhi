---

# 第1章 系统总体定位（必须）

回答：

```text
这个系统是干什么的？
```

建议包含：

---

## 1.1 系统目标

例如：

* 研究右侧趋势交易
* 研究量价行为
* 验证技术分析方法
* 评估策略可信度
* 沉淀研究知识

---

## 1.2 系统不做什么

这个很重要。

例如：

* 不做高频交易
* 不做自动实盘执行
* 不做黑盒AI预测
* 不直接输出“必赚策略”

这会避免认知偏差。

---

## 1.3 系统核心思想

建议重点写：

### ① 研究优先

先证明：
“这个方法真实有效”

再谈：
“怎么赚钱”

---

### ② Layer Q 审计

不是只看收益率。

而是：

* 是否稳定
* 是否跨周期
* 是否适应不同Regime
* 是否只是参数尖峰

---

### ③ 研究结果必须沉淀

系统不是“一次性回测”。

而是：

```text
研究 → 结论 → 知识积累 → 新研究
```

---

## 2.2 每一步在干什么（非常重要）

建议一层一层解释。

例如：

---

### 行情数据层

输入：

* stock_daily
* volume
* amount
* 龙虎榜
* VWAP
* 逐笔数据

作用：

```text
提供原始市场行为
```

---

### Factor因子层

作用：

```text
把原始行情转化为可分析指标
```

例如：

| 因子             | 用途    |
| -------------- | ----- |
| MA             | 趋势    |
| MACD           | 动量    |
| ATR            | 波动    |
| VWAP           | 成本    |
| OBV            | 资金流   |
| Volume Profile | 成交密集区 |

---

### Method方法层

作用：

```text
把因子转化为交易观点
```

例如：

```text
MACD金叉 → 看多
RSI超卖 → 反转
Wyckoff吸筹 → 主力建仓
```

---

### Signal层

作用：

```text
输出结构化研究结论
```

例如：

```json
{
  "direction": "BUY",
  "confidence": 0.78,
  "regime": "TREND_UP",
  "evidence": [...]
}
```

---

### Portfolio层

作用：

```text
决定资金如何配置
```

例如：

* 仓位
* 分散
* 风险预算

---

### Execution层

作用：

```text
模拟真实成交
```

例如：

* 滑点
* 冲击成本
* 流动性限制

---

### Layer Q层（重点）

作用：

```text
审计研究结果是否可信
```

例如：

* 是否过拟合
* 是否参数尖峰
* 是否跨周期失效
* 是否仅在牛市有效

---

### KnowledgeBridge层

作用：

```text
把研究结果沉淀成知识资产
```

例如：

* 最优参数
* 失效案例
* Regime适配
* 因子表现
