## 1. Stage 3 介入时机：墨萱通过后，不可并行

玄知架构把关**必须串行在墨萱回归对比+技术复核之后**，不可并行。

**原因**：
- 架构审查依赖墨萱的QA结论（偏差率、复现性验证结果）作为输入
- 若回归未通过（偏差过大），架构审查的前提不成立，应直接退回Stage 1
- 与墨枢四方会签发言顺序一致：墨萱 → 玄知 → 墨涵（不可跳步）

## 2. 架构把关需要墨衡提供的材料

| 材料 | 说明 |
|------|------|
| 黄金基线定义 | 预期输出（含通过标准）、基线生成方式 |
| 回归测试框架说明 | 测试范围、对比粒度、稳定性要求 |
| 核心模块接口文档 | 模块间数据流、关键函数签名与依赖关系 |
| 参数配置清单 | 回测参数、因子参数、随机种子（如适用） |
| 代码目录结构 | 模块划分、入口/出口文件 |

上述材料应内嵌在 Stage 1 的 `.done` 信号文件中，或附在黄金基线包的 README 内。

## 3. 流程同意确认

✅ **同意此5阶段编码流程**。

- Stage 1→2→3 串行，Stage 4 可与 Stage 3 末尾重叠（文档准备）
- Stage 3 输出：通过 / 有条件通过（附修改建议） / 不通过（退回Stage 1）
- 若"有条件通过"，墨衡实施修改后直接进入 Stage 2 增量回归，无需玄知二次把关
