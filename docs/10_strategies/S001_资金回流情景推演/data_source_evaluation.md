| 维度 | 评估 |
|:-----|:-----|
| **日线需求** | ❌ **不满足**。需要日级 OHLCV 作为回测基础数据 |

**结论**：CSV 不能直接用于 S001 滚动回测。本地 DB 的日线数据已足够覆盖 601857 的 12 个月滚动回测需求。

### 3.2 因子校准的补充验证价值

S001 的四重折扣因子链如下：
```
资金回流强度 = α₁(宏观) × α₂(行业) × α₃(动量) × α₄(结构)
```

#### α₂（行业折扣因子）— 市场情绪

| 评估维度 | 说明 |
|:---------|:-----|
| **相关性** | ⭐⭐⭐ 1 分钟频率可捕捉开盘/尾盘情绪变化、涨跌幅分布 |
| **充足性** | ⭐ 单日数据不足以做统计建模，但可作为**格式参考** |
| **建议** | 用于验证分钟级 α₂ 计算的可行性，不作为正式校准数据 |

#### α₃（动量折扣因子）

| 评估维度 | 说明 |
|:---------|:-----|
| **相关性** | ⭐⭐ 动量因子通常基于日/周频次，分钟级动量噪音大 |
| **充足性** | ⭐ 单日不足以构建动量信号 |

#### α₄（结构折扣因子）— 流动性与波动率

| 评估维度 | 说明 |
|:---------|:-----|
| **相关性** | ⭐⭐⭐⭐ **最相关**。分钟级数据可计算：日内波动率分布、成交量分布、换手率时序、买卖力量对比 |
| **充足性** | ⭐⭐ 单日可做样本典型日分析，但不能做分布统计 |
| **建议** | 可用于 α₄ 因子计算函数的**原型验证**，验证 1 分钟数据 → 日度流动性指标的计算链路 |

### 3.3 汇总评分

## 5. 结论与建议

### 5.1 核心结论

| 判断 | 结论 |
|:-----|:-----|
| **冗余 vs 增量** | ✅ **增量数据**。本地 DB 无 2025-07-21 任何记录 |
| **S001 回测可用性** | ❌ 无法满足 12 个月滚动回测的日线需求 |
| **S001 因子校准** | ⚠️ 有限补充验证价值，最适用于 α₄ 原型验证 |
| **S004 可用性** | ✅ 可直接作为微观结构测试数据 |

### 5.2 建议

1. **针对 S001 回测扩展**：建议通过 tushare API 补充更多标的（当前仅 3 只）的日线数据，而非依赖单日分钟数据
2. **1 分钟数据链路**：如果计划在未来将分钟级数据纳入回测体系（α₄ 的日内波动率），建议将此 CSV 作为**数据格式参考**，指导 `stock_minute` 表的分钟数据统一采集标准的制定
3. **S004 协同**：将该 CSV 传递给 S004 团队，用于微观结构分析管线测试
4. **日线缺失填补**：2025-07-21 的日线可通过 tushare 的 `pro.daily()` 补充一行数据，无需手动处理 CSV
