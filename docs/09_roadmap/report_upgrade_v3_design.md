## 二、P1 收益归因模块

### 2.1 算法说明

**目的**: 回答"收益从哪里来？"——分解收益到不同来源（来源层、时间层、方向层）。

**核心方法**:
1. **Brinson 归因（简化版）**: 将总收益分解为超额收益（交易决策贡献）+ 基准收益（买入持有）+ 残差
2. **时间维度分解**: 按月份、季度拆分收益贡献时间序
3. **交易维度分解**: 按交易方向（long/short）、持仓时长分组统计贡献

**计算公式**:

```
总收益 = Σ(每笔交易盈亏)

超额收益 = 总收益 - 基准买入持有收益

时间归因:
  月份贡献 = 当月已实现盈亏 / 总盈亏

方向归因:
  Long 贡献 = Σ(所有做多交易盈亏) / 总盈亏
  Short 贡献 = Σ(所有做空交易盈亏) / 总盈亏

持仓时长归因:
  短持有(≤5天)贡献 = Σ(短持有交易盈亏) / 总盈亏
  中持有(6~20天)贡献 = ...
  长持有(>20天)贡献 = ...
```

## 三、P2 风险归因模块

### 3.1 算法说明

**目的**: 回答"风险从哪里来？"——将策略风险分解到不同源头，识别主要风险贡献因子。

**核心方法**:
1. **夏普比率分解**: 将夏普比率拆解为收益部分和波动部分
2. **回撤归因**: 识别最大回撤的起止区间、回撤速度、恢复速度
3. **连亏/恢复时间分析**: 统计连续亏损笔数、恢复 Bar 数、连亏分布
4. **风险因子暴露（模拟）**: 估算策略对市场因子（Beta）、规模因子（Size）、价值因子（HML）、动量因子（MOM）的暴露度
5. **尾部风险估计**: 基于历史收益分布的 VaR(95%) / CVaR(95%) 估算

**计算公式**:

```
夏普分解:
  SR = (μ_annual - r_f) / σ_annual
  收益贡献 = μ_annual - r_f
  波动贡献 = σ_annual

回撤归因:
  最大回撤(D) = max(1 - 权益曲线_i / 前高_i)
  回撤速度 = 回撤深度 / 回撤期天数
  恢复速度 = 回撤深度 / 恢复期天数

尾部风险:
  VaR(95%) = 日收益率分布的 5% 分位数
  CVaR(95%) = 低于 VaR(95%) 的期望损失
```

#### 连亏/恢复时间分析

**目的**: 统计策略的连续亏损模式——连亏笔数、恢复时间、分布特征。连亏是策略真实的尾部风险来源（比最大回撤更精细）。

**核心指标**:
- 最大连续亏损笔数: 策略执行中连续亏损交易的最大笔数
- 平均恢复 Bar 数: 连亏发生后恢复到前高所需的平均 K 线数
- 最大恢复 Bar 数: 连亏发生后恢复到前高所需的最大 K 线数
- 连亏分布: 按连亏笔数分组的频次统计

**计算公式**:

```
最大连亏笔数 = max(consecutive_loss_sequence.length)
平均恢复 Bar 数 = mean(各次连亏后从最低点恢复到前高的 Bar 数)
最大恢复 Bar 数 = max(各次连亏后从最低点恢复到前高的 Bar 数)

连亏分布:
  连亏 1 笔频次 = count(连亏序列长度为 1)
  连亏 2 笔频次 = count(连亏序列长度为 2)
  连亏 3 笔频次 = count(连亏序列长度为 3)
  连亏 ≥4 笔频次 = count(连亏序列长度 ≥ 4)
```

## 四、P3 参数稳定性热力图模块

### 4.1 算法说明

**目的**: 量化分析参数选择对回测结果的影响——哪些参数敏感？哪些参数稳健？

**核心方法**:
1. **单参数边际效应**: 固定其他参数，观察单一参数变化对夏普/年化收益/最大回撤的影响
2. **双参数交互热力图**: 两两参数交叉，计算每个组合的绩效指标，生成热力图
3. **参数敏感性指数**: 量化每个参数对绩效指标的方差贡献
4. **最优解鲁棒性检验**: 最优参数 ±1 步长，结果是否大幅恶化？

**计算公式**:

```
单参数敏感性:
  S_p = Var(metric | p)  /  Var(metric | 所有参数)
  
  S_p 越大 → 参数 p 越敏感（越重要）

双参数热力图:
  M[i][j] = metric(参数A = i, 参数B = j)

  常见热力图层级: 夏普 / 年化收益 / 最大回撤 / 综合评分

鲁棒性检验:
  Δ_metric = metric(最优参数) - metric(最优 ± 1步长)
  
  Δ_metric 小 → 参数稳健
  Δ_metric 大 → 参数脆弱（过拟合风险高）
```

## 五、P4 Walk Forward 分析模块

### 5.1 算法说明

**目的**: 验证策略参数的泛化能力——是否在不同时间窗口内保持一致的表现？

**核心方法**:
1. **固定窗格 Walk Forward**: 将回测期划分为连续的训练期 + 测试期窗格
2. **滚动窗格 Walk Forward**: 滑动窗口划分（训练 60 天 + 测试 20 天 → 滚动前进）
3. **参数稳定性交叉验证**: 在前一个窗格选出的最优参数，在下一个窗格中表现如何？
4. **Walk Forward 效率比**: 测试期收益 / 训练期收益，比值接近 1.0 表示参数泛化良好

**窗格划分方案（回测期 84 天 → 循环组合期 2026-01-01 ~ 2026-05-14）**:

  训练: 01-01 ~ 02-08 (28天) → 测试: 02-09 ~ 03-18 (28天) → 测试2: 03-19 ~ 05-14 (28天)

方案 C（滚动窗格, 步长 20 天）:
  W1: 训练 01-01~02-08, 测试 02-09~02-28
  W2: 训练 01-21~02-28, 测试 03-01~03-20
  W3: 训练 02-08~03-20, 测试 03-21~04-09
  W4: 训练 02-28~04-09, 测试 04-10~04-29

## 六、P5 成交执行层模块

### 6.1 算法说明

**目的**: 评估回测的成交质量——滑点假设是否合理？市场冲击如何？限价单执行率？

**核心方法**:
1. **成交缺口分析**: 信号触发价格 vs 实际成交价格的差异（滑点实际值 vs 假设值）
2. **限价单执行率**: 限价单是否全部成交？未成交的比例？
3. **市场冲击估算**: 基于成交量的冲击模型（大单对小盘的冲击效应）
4. **成交时间延迟**: 信号触发 → 实际成交的时间间隔
5. **流动性分层**: 在低流动性时段（盘中/盘尾）的成交质量下降程度

**计算公式**:

```
执行缺口(Slippage):
  S_actual = P_executed / P_signal - 1
  S_default = 0.1% (当前系统假设值)
  S_差异 = S_actual - S_default  → 正数表示实际滑点大于假设

市场冲击:
  I = (V_trade / V_20d_avg) × 0.05%
  V_trade = 交易量(股数)
  V_20d_avg = 20日平均成交量

限价单执行率:
  ER = N_executed / N_limit_orders × 100%

流动性分层:
  High : Volume > 20d_MA × 1.5 → 执行良好
  Medium: Volume > 20d_MA × 0.7 → 正常执行  
  Low: Volume < 20d_MA × 0.7 → 执行质量下降
```

## 八、P7 因子 IC/RankIC 模块

### 8.1 算法说明

**目的**: 量化每个因子的预测能力——因子值与未来收益的相关性。

**核心方法**:
1. **IC (Information Coefficient)**: 因子值与未来 N 期收益的 Spearman 秩相关系数
2. **RankIC**: 因子值与收益排名的秩相关系数（鲁棒版）
3. **IC 时间序列**: 计算每日/每周的 IC 值，统计 IC 均值、标准差、T 统计量
4. **IC 衰减曲线**: 因子对未来 1/3/5/10/20 期收益的预测力衰减
5. **因子有效比例**: IC > 0 或 IC 显著（|t| > 2）的交易日占比

**可分析的因子列表**:

| 因子名称 | 定义 | 计算来源 | 当前可用 |
|:--------|:----|:--------|:--------|
| TrendQuality | 趋势质量评分(0~1) | TrendQuality 模块 | ✅ 已有 |
| Regime 虚拟变量 | TREND_UP=1, RANGE=0 | Regime 模块 | ✅ 已有 |
| VWAP 偏离度 | (价格 - VWAP) / VWAP | AnchoredVWAP | ✅ 已有 |
| 成交量变化率 | Volume / 20d_MA_Volume | Volume 模块 | ✅ 已有 |
| ATR 扩张比 | ATR / 20d_MA_ATR | VolatilityRiskManager | ✅ ATR 数据可用 |
| OBV 变化率 | (OBV - OBV_prev) / OBV_prev | OBV 因子（待实现） | ❌ 未实现 |
| 网格层曝光 | 当前触发层数 | 当前无（网格层状态） | ❌ 未记录 |

**计算公式**:

```
IC_t = rank(factor_value_t) 与 rank(future_return_{t+1:t+N}) 的 Spearman 相关系数

IC_avg = mean(IC_t)  # 平均 IC
IC_std = std(IC_t)   # IC 标准差
IR = IC_avg / IC_std # 信息比

IC_衰减(t) = IC(因子值, 未来 t 期收益)
  → t=1,3,5,10,20 的 IC 值序列 → 衰减曲线
```

## 九、P8 基准对比增强模块

### 9.1 算法说明

**目的**: 回答"策略比买入持有好吗？比同类型策略好多少？"

**核心方法**:
1. **买入持有对标基准**: 在相同资金 + 相同时期下的买入持有收益（Buy & Hold）
2. **大盘基准对标**: 601857 所属板块/指数（上证指数 / 石油石化板块 / 能源 ETF）
3. **同类策略对标（框架）**: 同类型网格策略在其他标的上表现
4. **超额收益检验**: 策略收益 - 基准收益的显著性检验（T-test / Bootstrap）
5. **超额收益分布**: 正超额交易占比、超额稳定性（一年中多少个月份跑赢基准）

**计算公式**:

```
买入持有收益:
  R_bh = (P_end / P_start - 1) × 100%

超额收益:
  α = R_strategy - R_benchmark

超额夏普:
  SR_α = (R_strategy - R_bh) / σ_超额收益

超额稳定性:
  Stability = 跑赢基准月份 / 总月份 × 100%
  
  注：当前 84 天约 4 个月，显著性有限
```
