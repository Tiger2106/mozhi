**P25"798条"表述的验证：**
- "798条"指 `pe_ttm` 因子在 `a50_cross_ic_result` 表中的记录数。查询确认 `pe_ttm` 共 **798条**记录，与表述一致。

- v1 是唯一被管线(marine-s/mellow-o)使用的 source_version。所有三次运行（marine-s、mellow-o、--no-run）以及验证脚本查询均使用 v1。

- 但可通过 `created_at` 时间戳格式区分：管线正式运行使用ISO格式 `+08:00`，而批处理/再处理使用本地格式 `YYYY-MM-DD HH:MM:SS`。

**结论：** 已查证。口径问题确认存在并已定位到具体范围（798条 = pe_ttm 在最终静态库的记录数，时间点应为19:55而非~18:12）。

## 交叉查证发现

### 发现1：口径问题确认（关联H1）
P25"798条"的标注时间点"~18:12"不准确。18:12快照时实际只有部分数据写入完成。798条对应的是**最终静态库**（~19:55），而非18:12快照。

### 发现4：source_version 的区分能力
管线正式运行：`source_version='v1'` + `created_at` ISO格式（含 `+08:00` 后缀）
