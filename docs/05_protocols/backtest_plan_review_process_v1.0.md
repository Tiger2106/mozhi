# 回测试验方案评审会流程 v1.0

## 定位

本流程覆盖**试验方案执行前的评审阶段**，即模板中的 §0（R001 问题定义）→ 设计方案完成 → §0.5 审批流程。执行完成后的结果评审，则走已存在的《回测总结会4阶段流程 v1.1》。

---

## 总体结构

```
        ① 方案准备                    ② 四级会签                    ③ 准入 / 终止
                                           ↓
    ┌─────────────────┐          ┌─────────────────┐         ┌──────────────────────┐
    │  墨涵主持 R001  │          │  墨萱技术签      │         │  Owner 终签           │
    │  问题定义四步    │ ──→     │  §2/3/6 合规核验  │ ──→    │  准予执行 / 终止      │
    │  团队共识放行    │          │  回退≤2次        │         │                       │
    └─────────────────┘          └─────────────────┘         └──────────────────────┘
                                        ↓                              ↓
                                 ┌─────────────────┐           ┌──────────────────────┐
                                 │  墨涵知识签      │           │  version_status 更新  │
                                 │  §0+R001 归档核验 │ ←─── ───│  草案→准予执行/已归档  │
                                 │  不代修退回      │           └──────────────────────┘
                                 └─────────────────┘
```

## 与回测总结会的定位差异

| 维度 | 方案评审会（本流程） | 回测总结会（已存在 v1.1） |
|:----|:-------------------|:------------------------|
| 评审对象 | 试验方案（执行前） | 试验结果（执行后） |
| 输入 | 填写完毕的 EXP 模板（§0~§7 草案状态） | TMPL-001/002/003 报告 |
| 输出 | 准予执行 / 退回修改 / 终止 | 归档入库 / 知识沉淀 |
| 核心判定 | 设计是否合理+可行 | 结果是否正确+可重复 |
| 退回模式 | 不代修，退回起草人 | 不代修，退回墨衡修复 |
| 耗时上限 | 30 分钟（含 R001） | 40 分钟 |

## Phase 1：方案准备（R001 问题定义）

**主持人：** 墨涵
**参与者：** 全员（提案人必到，其余按需）
**时长：** 随问题复杂度，但墨涵在争议超过30分钟时须触发船东裁决

### Phase 1 流程

```text
墨涵主持：四步引导
  Step 1  描述现状（提案人陈述）
           - 现象是什么？有数据支撑
  Step 2  分析后果（全员补充）
           - 影响范围？紧急性？
  Step 3  寻找根因（至少追问三层）
           - 第一层：直接原因
           - 第二层：机制原因
           - 第三层：系统/认知原因（或标注"已追至可解释极限"）
  Step 4  统一问题陈述（墨涵记录）
           - 格式要求："对象 + 偏差 + 可观测指标"
           - 全员确认后方可进入下一阶段
```

### R001 输出

```json
{
  "status": "共识确认",
  "statement": "l_vol_atr在2024年后信号与净值出现背离，IS IR=0.08暴跌至OOS IR=-0.02",
  "root_cause": "第三层：市场状态切换（低波→高波）导致波动率因子方向性反转",
  "observable": "rolling_3m_IR <-0.02或IR/IQ比率<0",
  "confirmed_by": ["墨衡", "墨萱", "玄知", "Owner"]
}
```

R001 完成后，墨涵将问题陈述写入试验模板 §0，version_status 推进至"草案"。

## Phase 2：设计方案审核（四轮逐级会签）

**原则：** 顺序固定，不可并联。墨萱签 → 墨涵签 → Owner签。

### 第1轮：墨萱技术签

**核验范围（对应模板节次）：**

| 核验项 | 对应节 | 通过判定 |
|:------|:-----|:---------|
| 标的排除标准合规（上市≥2年、free_float≥0.5、复权质量） | §2.1 | 全部满足或已标注例外说明 |
| 数据精度检验规则是否明确 | §2.3 | data_qc 脚本路径已指定，assert 已替换为 raise 硬阻断 |
| 因子权重配置无循环依赖 | §3.1 | factor_weight 与 factor_signal 已分离 |
| A股硬约束已启用（最小交易单位1手、印花税、价格笼子） | §3.2 | 已标注启用/禁用状态 |
| 停复牌运行规则已定义（持有/信号保留/复牌价格/异常标记） | §2.2 | 已完整填写 |
| IC/IR 验收阈值已设定（>50%正值比硬约束，>53%优秀线） | §4.1 | 已填入对应数值 |
| P2 自洽性检查已规划（固定seed、预留自洽性验证时间） | §2.3 | 已标注 |

**判定：**
- 全部通过 → 进入第2轮
- 不通过 → 退回起草人修改，附具体修改要求（空退回无效）

### 第2轮：墨涵知识签

**核验范围：**

| 核验项 | 通过判定 |
|:------|:---------|
| R001 问题陈述完整（对象+偏差+可观测指标） | 三者齐备 |
| 模板 §0 已填写（问题记录、根因分析、全员确认签名） | 无缺失 |
| 失败案例库已自查（knowledge.db 已查询，cold_start 可放行） | 已执行或注明冷启动 |
| 方案设计是否合理、修改方向是否正确 | 符合修改要求 |

**退回规则（不代修）：**
- 不合格 → 退回原起草人，注明：**不合格范围 + 修改方向**
- 不直接在文件上代修（格式/结构错误除外）
- 起草人修改后重新提交，墨涵再次判定

### 第3轮：Owner 终签

**核验内容：**
- 业务方向确认：方案是否值得投入资源执行
- 可行性确认：在现有算力和数据条件下是否可执行

**判定：**
- **准予执行** → version_status 推进至"准予执行"，进入执行阶段
- **终止** → version_status 标记为"已终止"，按失败归档流程处理

### 第4轮：后续备案（墨涵执行，无需会议）

会签全部通过后，墨涵完成：
1. version_status 从"评审中"推进至"准予执行"
2. 会签结果写入 `backtest_run.notes`
3. 试验编号 EXP-{YYYY}-{XXX} 正式激活

## 退回与裁决机制

### 退回循环

```text
第1次退回
  └── 签字方 → 退回起草人（附具体修改要求）
      └── 起草人修改 → 重新提交 → 签字方判定
          ├── 通过 → 进入下一轮会签
          └── 不通过 → 第2次退回

第2次退回
  └── 签字方 → 退回起草人（加注更具体要求）
      └── 起草人修改 → 重新提交 → 签字方判定
          ├── 通过 → 进入下一轮会签
          └── 不通过 → 强制触发船东裁决

第3次（船东裁决）
  └── Owner 判定："通过"或"终止"，结论不可再裁决
      └── 结果写入 backtest_run.notes
```

## 缩略流程（共识已达成时的快速通道）

当以下条件全部满足时，可跳过 Phase 1 直接进入 Phase 2：
- 提案人与 Owner 已在会前达成 R001 共识（有书面记录）
- 修改项仅涉及数值调整（非设计重构）
- 无跨团队依赖项

**不可跳过：** 退回循环（第1次/第2次）和船东裁决在任何情况下不可简化。
