## 1. 格式规范

### 1.1 historical_reference_{case_id}.json 格式

```json
{
  "case_id": "<策略案例唯一标识>",
  "time_range_start": "<YYYY-MM-DD>",
  "time_range_end": "<YYYY-MM-DD>",
  "symbols": ["<标的代码1>", "<标的代码2>"],
  "pattern_match_score": "<相似度评分 0~1>",
  "notes": "<额外备注/补充说明>"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|:----|:----|:----:|:----|
| case_id | string | ✅ | 策略案例唯一标识，如 "S003_HIST_001" |
| time_range_start | string(YYYY-MM-DD) | ✅ | 历史案例起始日期 |
| time_range_end | string(YYYY-MM-DD) | ✅ | 历史案例结束日期 |
| symbols | string[] | ✅ | 涉及标的代码列表 |
| pattern_match_score | float[0,1] | ✅ | 与当前策略的相似度评分 |
| notes | string | ✅ | 额外补充说明或备注 |

### 1.2 格式对齐确认

本格式与 S002 格式规范完全对齐：
- 语义一致：case_id 对应历史案例记录；time_range 标识回溯窗口
- 结构一致：均为扁平 JSON 对象，无嵌套冗余
- 类型一致：symbols 为数组，score 为浮点数，符合 S002 规范
