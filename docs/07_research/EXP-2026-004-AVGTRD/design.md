## §0 问题定义（R001 四方会签）

### 现状

- avg_trade_price = amount/volume，基础数据质量好（0 NULL、0 零值）✅
- 尚未入库因子库，未做任何 IC/IR 检验 ✅
- 名义价格偏差严重（茅台1,660元 vs 石油7元），截面方差来自股价本身 ✅
- 前置条件：free_float 以 tushare pro 为锁定口径，当前 free_float_share 100% NULL
- 数据源以 market_data.db（619MB）为准：50只全量、2007~2026 完整覆盖 ✅

### 后果分析

1. AVGTRD-004 试验直接阻塞，无法 bypass
2. 因子库缺 avg_trade_price 列，影响完整性
3. EXP-005（双量L2）缺乏 IC 基准参考
4. free_float_share 100% NULL 是 P0 系统级阻塞——所有依赖流通市值口径的回测均不可用，影响远超本试验
5. 紧急性高——窗口期敏感，错过会被后续试验挤压

### 根因

1. **L1 执行层**：avg_trade_price 从未入库因子库，排期上被 EXP-001~003 推后
2. **L2 流程层**：free_float NULL 阻塞 pipeline 入口，人工检查清单未覆盖
3. **L3 数据基建层**：tushare pro free_float 接口从未采集，且系统缺少截面标准化层
4. **L4 治理层**：系统缺少"依赖图自动解析→发现缺口→触发采集"机制，前 3 个试验都不需要 free_float，缺口一直被隐藏

## §2 试验目标

### 2.1 核心问题

> **avg_trade_price（平均成交价）作为 L1 绝对量因子，IC 方向是否稳定正向？在不同市场状态下是否存在显著分化？**

### 2.2 停复牌与异常数据处理规则

**停牌数据处理：**
- 停牌 stock forward_return 计算：当前交易日停牌时，延长持有窗口至复牌日（即 T 日停牌 → 实际计算 T+suspend_days 后的首个交易日收益）
- 复牌首日数据保留（复牌日成交量通常显著放大，属于正常交易行为，不排除）
- 连续停牌超过 20 个交易日的标的，从回测标的池中移除（避免因长时间停牌导致 forward_return 计算窗口异常拉长）
- Q4 分层回测中，停牌日的标的维持原分组不变（不主动调出），但当日不计算该标的的收益贡献

**零成交量处理：**
- volume=0 的交易日样本直接过滤，不在因子值计算中纳入
- forward_return 计算涉及停牌日的，按上述规则处理

**涨跌停数据处理：**
- 涨停日（pct_chg ≥ 9.9%）不可买入（forward_return 计算中，该日的买入信号触发但不得实际分配仓位）
- 跌停日（pct_chg ≤ -9.9%）不可卖出（同理，卖出信号触发但不得实际执行）

### 2.3 数据精度检验规则

因子计算与数据清洗阶段需通过以下精度检验，全部通过后方可进行后续分析：

| # | 检验项 | 方法 | 通过标准 |
|:-:|:-------|:-----|:---------|
| 1 | amount/volume 比值异常值检测 | 计算日均 amount/volume 比值，偏离 3σ 的交易日标记为异常 | 异常样本数 ≤ 总样本的 1%，否则需排查数据源 |
| 2 | 复权因子完整性检查 | 检查 adj_factor 是否存在断点（NULL 或每日变动 > 10%） | 无断点，日均变动 ≤ 10% |
| 3 | ETL 行数校验 | 源表（stock_daily）与计算后的因子表行数核对，差值 ≤ 0.1% 为通过 | 行数差异率 ≤ 0.1% |
| 4 | 采样核查 | 随机抽取 30 个交易日，手算 avg_trade_price 与自动计算结果比对 | 全部一致（计算逻辑无歧义） |
| 5 | 因子值边界检验 | 检查 avg_trade_price 是否在合理区间（最低价 ≤ avg_trade_price ≤ 最高价 × 1.1） | 超出该区间的样本 ≤ 总样本的 0.1% |

### 2.4 子问题分解

| 优先级 | 子问题 | 目标 | 说明 |
|:------:|:-------|:-----|:------|
| **P0 — Q1** | **IC 方向稳定性** | 检验 avg_trade_price 在日频截面上的 Spearman Rank IC 方向是否显著正向，在训练期与验证期的方向是否一致 | 首次因子检验，确认方向是核心目标 |
| **P1 — Q2** | **统计显著性验证** | Bootstrap + FDR BH 校正判断 IC 是否统计显著（非噪声） | 排除偶然相关 |
| **P2 — Q3** | **市场状态分化** | 按波动率/流动性分组，检验 avg_trade_price 在不同市场状态下 IC 是否分化 | 为后续因子使用场景提供依据 |
| **P3 — Q4** | **分层回测验证** | 按 avg_trade_price 分 5 层组合，检验单层收益单调性 | 确认因子选股能力不依赖 IC 统计量 |

### 2.5 Q1 工作包详述

**子目标**：确认 avg_trade_price 在 A50 全池日频截面上的 IC 方向。

| 检验项 | 方法 | 预期 |
|:-------|:-----|:------|
| 全样本 IC 均值 | Spearman Rank IC（日频） | 确认正/负/不显著 |
| IC 时序标准差 | 日度 IC 序列的标准差 | 评估波动性 |
| ICIR 计算 | 均值 IC / 标准差 IC | 评估信号强度与一致性 |
| 正向天数占比 | IC>0 的交易日占比 | 辅助判断方向稳定性 |

**补充说明**：avg_trade_price = amount/volume，从量价关系逻辑来看：
- 高平均成交价通常对应大资金主动买入（推动价格上涨），可能对未来收益有正向预测力
- 低平均成交价可能对应被动交易或卖盘压力，预期收益偏弱
- 但该因子作为 L1 绝对量因子，受个股价格绝对水平影响（高价股自然 avg_trade_price 高），需要验证名义价格水平带来的截面偏差

### 2.6 Q2 工作包详述

**子目标**：通过严格的统计检验确认 IC 显著性。

| 检验项 | 方法 | 通过标准 |
|:-------|:-----|:---------|
| Bootstrap IC 分布 | 排列因子值，1000 次抽样 | 观测 IC 在排列分布中处于置信区间外 |
| FDR BH 校正 | 多重比较校正，q=0.05 | 显著因子比例 q < 0.05 |
| L3 稳定性检验 | 四维度（时段/标的/参数/方法）稳定性 | 4/4 通过 |

### 2.7 Q3 工作包详述

**子目标**：检验 avg_trade_price 在不同市场状态下的 IC 分化。

| 状态分组 | 定义 | 预期 |
|:---------|:-----|:------|
| 高波动期 | 标的 20d 滚动波动率 > 30% 分位数 | 资金行为在高波动期更激进，IC 可能更强 |
| 低波动期 | 标的 20d 滚动波动率 < 30% 分位数 | 低波动期交易不活跃，IC 可能减弱 |
| 高成交期 | 标的 20d 滚动换手率 > 70% 分位数 | 高活跃度下有更充分的价格发现 |
| 低成交期 | 标的 20d 滚动换手率 < 30% 分位数 | 低活跃度下价格可能失真 |
| 上涨/下跌市 | 按 A50 指数涨跌 2% 分界 | 上涨市 vs 下跌市的分化情况 |

**预期结论假设**：
- avg_trade_price 在高波动/高成交期间可能呈现更强的正向 IC
- 在低波动/低成交期间 IC 可能衰减甚至翻转
- 上涨市 vs 下跌市的分化需验证（市场方向 vs 资金行为方向的关系）

### 2.8 Q4 工作包详述

**子目标**：通过分层回测确认因子实际选股能力。

| 探索项 | 方法 | 说明 |
|:-------|:-----|:------|
| 五分位分组 | 按 avg_trade_price 从低到高分 5 组，每日重平衡 | 检验收益单调性 |
| 多空组合 | Top 组 - Bottom 组（Long-Short 组合） | 确认因子是否能产生稳定的多空收益 |
| 行业中性化 | 组内减去行业均值后重新排序 | 检验收益是否独立于行业暴露 |

## §3 回测参数

### 3.1 基础参数

| 参数 | 值 | 说明 |
|:----|:---|:------|
| **标的池** | **A50 全池（50只）** | 含全部 A50 成分股（当前所有 50 只，与 EXP-003 保持一致）<br><br>**标的入选排除标准：**<br>1. 上市时间 ≥ 2 年（从 stock_basic.list_date 计算，排除次新股）<br>2. free_float_share / 总股本 ≥ 0.5（流通盘占比≥50%，依赖 free_float 数据采集完成后启用）<br>3. 复权因子连续无异常（adj_factor 日均变动 ≤ 10%，无断点）<br>4. 排除连续停牌超过 20 个交易日的标的（参见 §2.2） |
| **时间窗口** | **独立窗口** | 训练期 2007~2019 → 验证期 2020~2026-04 |
| **训练期** | 2007-01-04 ~ 2019-12-31 | 因子参数估计、阈值锁定、IC 基准计算 |
| **验证期** | 2020-01-01 ~ 2026-04-30 | 独立验证，无未来信息泄漏 |
| **数据频率** | 日线 | stock_daily 表日频数据 |
| **持有期** | **1 日 / 5 日 / 20 日** | 三档持有期对比；主评估 20 日（与 EXP-003 成本约束一致） |
| **因子计算频率** | T 日收盘后计算 avg_trade_price(T) | 用于预测 T+1 至 T+持有期的收益 |

### 3.2 因子定义

| 字段 | 定义 |
|:----|:------|
| **因子名称** | avg_trade_price（平均成交价） |
| **计算公式** | avg_trade_price = amount / volume |
| **amount** | 日成交金额（元），stock_daily.amount |
| **volume** | 日成交量（股），stock_daily.volume |
| **单位** | 元/股（每股平均成交价格） |
| **数据类型** | L1 绝对量因子 |
| **计算时点** | 每日收盘后 |
| **入库状态** | ❌ 尚未入库因子库，需新建因子表 |

### 3.3 研究方法

| 方法 | 参数 | 说明 |
|:----|:-----|:------|
| **IC 计算** | Spearman Rank IC（日频截面） | 因变量为持有期收益率（1d/5d/20d forward return） |
| **Bootstrap** | 排列因子值，1000 次 | 零假设：IC=0，检验观测 IC 是否在排列分布尾部 |
| **多重比较校正** | **FDR BH q=0.05** | 替代 Bonferroni（更优的 I 型/II 型错误平衡） |
| **L3 稳定性** | 4/4 通过 | 四个维度稳定性检验：时段切分 / 标的子集 / 参数偏移 / 方法变体 |
| **分层回测** | 五分位等权组合、行业中性化 | 验证收益单调性与多空组合收益 |
| **敏感性分析** | 按时间窗口切分（每 3 年一个子窗口） | 检验 IC 在子窗口中的方向一致性 |

### 3.4 交易成本约束

| 方向 | 佣金 | 印花税 | 滑点 | **合计** |
|:----:|:----:|:------:|:----:|:--------:|
| 买入 | 0.03% | 0% | 0.10% | **0.13%** |
| 卖出 | 0.03% | 0.05% | 0.10% | **0.18%** |
| **单次完整交易（买+卖）** | — | — | — | **0.31%** |

> **持有期成本对照**：
> - 20 日持有期：年化约 12.6 次换手，年化成本拖累 ~4%
> - 5 日持有期：年化约 50.4 次换手，年化成本拖累 ~15%
> - 1 日持有期：年化约 252 次换手，年化成本拖累 ~78%
>
> 主评估以 20 日持有期为准，短期持有期结果需附带显著成本警示。

### 3.5 A股市场约束

| 约束类型 | 规则说明 | 影响范围 |
|:---------|:---------|:---------|
| **涨跌停板约束** | 涨停日（pct_chg ≥ 9.9%）不可买入，跌停日（pct_chg ≤ -9.9%）不可卖出 | 影响 forward_return 计算中的开仓/平仓时点判定；Q4 多空组合中，若 Top 组标的涨停则不可追加买入，Bottom 组标的跌停则不可追加卖出 |
| **T+1 交易约束** | 当日买入不可当日卖出，隔日方可平仓 | 1 日持有期策略受此约束最大（实际最小持有期为 T 日买入 → T+1 日卖出），在回测中 1d forward_return 计算起点应为 T+1 收盘而非 T+1 开盘 |
| **卖空约束** | A50 标的均为融资融券标的，但融券存在成本和券源限制 | Q4 多空组合中空头端需考虑：融券年化费率约 8~10%（需纳入成本模型），券源不足时可能导致空头仓位无法足额配置，建议在 Q4 分析中注明"空头端受融资融券约束，实际可执行性低于多头端" |

**各工作包受影响说明：**
- **Q1（IC 方向）**：涨跌停影响截面 IC 计算的代表性（涨停日部分标的 IC 值被截断），需在 IC 计算时剔除当日涨停/跌停的标的
- **Q4（分层回测）**：多空组合受卖空约束影响最大，实际收益可能低于理论收益，需在结果中单独标注空头约束提示

## §4 评估标准

### 4.1 核心评估维度

| 维度 | 指标 | 通过条件 |
|:-----|:-----|:---------|
| **IC 方向** | Spearman Rank IC | 全样本 IC > 0，与训练期方向一致 |
| **统计显著性** | FDR BH q < 0.05 | 验证期保持显著 |
| **跨窗口衰减** | 训练期 IC → 验证期 IC 衰减率 | 衰减率 ≤ 50%（判定为稳健） |
| **ICIR** | 均值 IC / 标准差 IC | ICIR > 0.3（弱信号线）|

### 4.2 Q1 评估标准

| 判定 | 条件 |
|:----|:------|
| **PASS** | 全样本 IC 方向为正（或预期方向）且与训练期一致，ICIR > 0.3 |
| **WARN** | 方向一致但 ICIR ≤ 0.3（信号弱但不翻车）|
| **FAIL** | 方向不一致，或全样本 IC 绝对值 < 0.005（基本无信号）|

### 4.3 Q2 评估标准

| 判定 | 条件 |
|:----|:------|
| **PASS** | Bootstrap p < 0.05，FDR BH q < 0.05，L3 4/4 通过 |
| **WARN** | Bootstrap p < 0.05 但 FDR BH q ≥ 0.05，或 L3 3/4 通过 |
| **FAIL** | Bootstrap p ≥ 0.05（IC 不显著）|

### 4.4 Q3 评估标准

| 状态 | 指标 | 目标值 |
|:-----|:-----|:-------|
| 高波动 vs 低波动 | 两组 IC 差值 | 若差值 > 0.03，判定为存在分化 |
| 高成交 vs 低成交 | 两组 IC 差值 | 若差值 > 0.03，判定为存在分化 |
| 上涨市 vs 下跌市 | 两组 IC 差值 | 若差值 > 0.03，判定为存在分化 |
| 方向一致性 | 各子组 IC 方向 | 所有子组 IC 方向一致为正（无翻转）|

### 4.5 Q4 评估标准

| 指标 | 目标值 |
|:-----|:-------|
| 五分位收益单调性 | 低到高 5 组收益严格单调 |
| 多空组合夏普 | 年化夏普 > 0.5（考虑成本后） |
| 行业中性化收益 | 中性化后收益方向不变 |

### 4.6 评估方法清单（方案评审用）

| 检验项 | 方法 | 通过条件 | 是否已确定 |
|:-------|:-----|:---------|:----------:|
| IC 方向检验 | Spearman Rank IC | 方向一致 | ✅ |
| Bootstrap 显著性 | 1000 次排列 | p < 0.05 | ✅ |
| 多重比较校正 | FDR BH q=0.05 | q < 0.05 | ✅ |
| 稳定性集成 | L3 四层次检验 | 4/4 通过 | ✅ |
| 敏感性分析 | 时间窗口切分 | IC 方向不翻转 | ✅ |
| 分层回测 | 五分位 + 多空组合 | 收益单调 + 夏普 > 0.5 | ✅ |

## §5 数据需求与入库计划

### 5.1 数据源

| 数据源 | 表名 | 所需字段 | 状态 |
|:-------|:-----|:---------|:----:|
| **market_data.db** | stock_daily | ts_code, trade_date, amount, volume, pct_chg, close, adj_factor | ✅ 已就绪 |
| **因子库** | TBD（新建） | ts_code, trade_date, avg_trade_price | ❌ 需新建 |
| **tushare pro** | daily_basic | ts_code, trade_date, free_share → free_float_share | ✅ 2020~2026 已采集（77,148 行）<br>❌ 2007~2019 待补

### 5.2 字段计算说明

| 字段 | 来源 | SQL 计算 |
|:-----|:-----|:---------|
| avg_trade_price | amount / volume | `CAST(amount AS REAL) / CAST(volume AS REAL)` |
| forward_return_1d | close(T+1) / close(T) - 1 | 需要使用复权因子调整 |
| forward_return_5d | close(T+5) / close(T) - 1 | 5 日持有期收益 |
| forward_return_20d | close(T+20) / close(T) - 1 | 20 日持有期收益 |
| 复权因子 | stock_daily.adj_factor | 价格复权处理 |

### 5.3 入库计划（因子库）

| 操作 | 说明 | 优先级 |
|:-----|:------|:------:|
| 新建 avg_trade_price 因子表 | 在因子库中创建字段定义 | P0（试验开始前完成） |
| 历史批量计算（2007-01-04 ~ 2026-04-30） | 一次性回填全量因子值 | P0 |
| 因子值一致性校验 | 与原始数据比对，确保计算公式正确 | P1 |
| 日更管线接入 | 若试验通过，接入 daily pipeline 定时计算 | 待定（取决于试验结论） |
| **free_float 历史数据补采（2007~2019）** | 补全 A50 全量历史 free_float_share | **P0**（回测 pipeline 入口阻塞项）

### 5.4 数据依赖风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| amount/volume 字段存在缺失值 | 因子值缺失 | 缺失值不做填充，仅使用有效数据计算 |
| 复权因子影响 forward return 计算 | 收益率计算偏差 | 使用 adj_factor 复权计算收益率 |
| 零成交量日（停牌/涨停） | 因子值分母为 0 | 过滤 volume=0 的样本 |

## §8 预计工作量（墨衡估算）

### 8.1 按工作包分解

| 工作包 | 内容 | 预估耗时 | 说明 |
|:------:|:-----|:--------:|:------|
| **Q1** | IC 方向稳定性 + ICIR | **2h** | SQL 数据提取、IC 计算、分窗口统计 |
| **Q2** | Bootstrap + FDR + L3 | **3h** | Bootstrap 循环（1000 次）、多重比较、L3 维度展开 |
| **Q3** | 市场状态分组 IC | **2h** | 波动率/成交量/指数方向分组计算与对比 |
| **Q4** | 分层回测 | **3h** | 五分位分组、多空组合、行业中性化 |
| **数据准备** | 因子值入库 + 复权收益率 | **2h** | 因子表创建、批量计算、一致性校验 |
| **free_float 采集（数据基建）** | tushare pro daily_basic 采集 2007~2019 历史补全 | **1h** | 2020~2026 已完成，2007~2019 待补 |
| **文档/报告** | 结果汇总 + 知识库回填 | **1h** | 结论整理、回填触发 |
| **缓冲** | 问题排查+Edge case 处理 | **1h** | 缺失值、边界情况处理 |
| | **合计** | **~15h** | 约 2 个工作日 |

### 8.2 关键路径

```
数据准备 (2h) → Q1 IC 检验 (2h) → Q2 显著性 (3h) → [Gate: Q1+Q2通过?] → Q3 状态分化 (2h) → Q4 分层回测 (3h) → 文档 (1h)
```

**Gate 说明**：若 Q1+Q2 结论为 FAIL（IC 方向不稳定或不显著），Q3/Q4 不做深入研究，仅补充简要说明后直接汇总。避免在无效因子上过度投入。

- **Git 状态**：首次创建，初始 commit
- **可复用组件**：前视偏差防护、Bootstrap、FDR 函数可复用 EXP-003 已实现的代码框架
