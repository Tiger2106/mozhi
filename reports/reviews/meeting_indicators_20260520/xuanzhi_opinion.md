> **审查时间：** 2026-05-20T20:48+08:00

## ═════════════════════ 因子1：成交额均价（amount / volume）═════════════════════

### 1. 研究价值评估：高

**核心价值——补充了现有量价因子族的一个重要缺口。**

现有 VWAP 因子回答的是"一段时间内的加权平均成本"（依赖 typical_price 估算），成交额均价回答的是"今天市场实际成交的平均价格"。两者互补而非冗余，具体原因：

| 维度 | 现有 VWAP | 成交额均价 |
|:-----|:----------|:-----------|
| 数据来源 | typical_price 估算 | 交易所汇总的实际成交额/成交量 |
| 精度 | 近似值（bar 级估算） | 精确值（交易所统计） |
| 窗口依赖 | 需要多周期累计 | 单日即可计算 |
| 信号含义 | 机构持仓成本线（长期） | 当日实际成交均价（短期） |

**更重要的是：两者偏差本身就是一个有价值的价格结构信号。** 若成交额均价显著高于 VWAP，说明今日成交在日内高位区间更活跃（买方主导）；反之说明低位区间活跃（卖方主导）。这个偏差信号对于基金行为研究方法（指南§9.2，最高优先级方向）有直接输入价值——大单资金往往推动成交额均价偏离 VWAP。

### 2. 优先级定位：P0 — 与指南§9 框架一致

| 指南§9 优先级方向 | 与本因子的关联 | 对齐程度 |
|:-----------------|:--------------|:---------|
| 9.2 基金行为研究（最高优先级） | 成交额均价 vs VWAP 的偏差是检测大单资金进入的早期信号 | ✅ 强关联，直接输入 |
| 9.3 多Regime动态策略 | 可用于量价结构维度的 Regime 特征刻画 | ✅ 中等关联 |
| 9.4 策略组合 | 作为新增量价因子进入策略相关性矩阵 | ✅ 基础数据补充 |

**结论：因子1 是正在推进的§9.2 基金行为研究的"前置基础设施"之一，属于优先方向的基础铺路工作，优先级 P0 合理。**

### 3. 与 §12 路线图对齐

| §12 里程碑 | 对齐关系 | 说明 |
|:-----------|:---------|:------|
| v1.5 多策略组合 + RegimeRouter | 新增因子 → 更丰富的 factor 族 → 更精确的 Regime 特征 → 更好的 RegimeRouter 输入 | 间接但可追溯 |
| Q3~Q4 量价结构完善 | Wyckoff 量化验证需要精确的实际成交均价（非估算值） | 直接输入 |

**对齐评级：中度对齐** — 不是路线图中任一里程碑的直接触发条件，但为后续基金行为研究和 RegimeRouter 提供了基础数据支撑。

### 4. 复杂度评估：简单

| 评估项 | 结论 |
|:-------|:-----|
| 数据依赖 | 零新增，amount/volume 已存在 |
| 计算逻辑 | 一步除法，4 行代码 |
| 风险等级 | 🟢 低 |
| 测试工作量 | 少量可接受 |
| 墨衡估算 | 0.5 人天 ✅ |
| 墨萱估算 | 0.5 人天（含测试）✅ |
| **玄知评级** | **简单** |

### 5. 边界条件（墨衡+墨萱未覆盖项）

**无重大遗漏。** 墨萱已补充了墨衡遗漏的 amount/volume 单位一致性验证。玄知从研究价值角度增加一条：

**边际注意事项：** 成交额均价与 close 价格的偏离度可以作为情绪指标。若当日成交均价远高于收盘价（即价格冲高回落后收盘在低位），可能暗示日内高位追多资金被套——这是次日潜在卖压的来源。这一步属于因子用途拓展，不影响当前实现方案，可在文档中注明。

## ═════════════════════ 因子2：实际换手率（turnover_rate）═════════════════════

### 1. 研究价值评估：高

**核心价值——填补了当前因子族在"流动性维度"的系统性空白。**

指南§12 中明确标注了"Regime 研究"和"容量测试"是两个核心方向。换手率因子直接服务于这两者：

**对 Regime 研究（§9.3）的意义：**
- 换手率的剧烈变化是 Regime 切换的核心前置信号之一
  - 换手率骤升 + 价格横盘 → 可能的顶部换手
  - 换手率骤降 → 流动性枯竭，可能向 PANIC 或 LOW_VOL 转换
- 缺少换手率，Regime 分类的输入空间缺少了最关键的"量"的维度

**对容量研究（§12 Layer Q Q4）的意义：**
- 换手率是判断标的流动性的直接指标
- 流动性 → 可容纳资金规模（Q4 容量上限评估）和滑点成本
- 墨萱报告中已经提到：换手率可作为 Q5（TemporalValidator）的输入参数

**从基金行为研究角度（§9.2）：**
- 机构建仓/减仓往往伴随换手率的结构性变化
- 大单进入期间换手率异常升高是判断机构行为的重要佐证

**研究价值综合判断：**
| 受益方向 | 价值类型 | 重要程度 |
|:---------|:---------|:---------|
| Regime 研究（§9.3，P0） | 输入因子 | 不可或缺 |
| 基金行为研究（§9.2，P0） | 佐证因子 | 重要 |
| Lay Q Q4 容量评估 | 基础指标 | 必要 |
| 信号衰减研究（§9.6） | 流动性变化追踪 | 中等 |

### 2. 优先级定位：P0 — 但建议串行顺序上排在因子1之后

**P0 的理由：**
- 换手率是 Regime 分类器的重要输入维度
- §12 路线图中 v1.5（2026 Q3）要求 Regime 切换检测准确率 ≥ 80%，缺少换手率特征很难达到这个准确率
- 紧耦合于正在推进的基金行为研究

**建议串行顺序：因子1 → 因子2**

理由：
- 因子1 无数据依赖风险，极低复杂度，可作为"热身任务"快速完成
- 因子2 存在数据层清理风险（DB 中 3080 条 turnover_rate=0.0），墨萱已指出需要在阶段一先清空
- 因子1 快速上线，可立即用于辅助验证因子2 的实现正确性（两者都是 volume 族因子）

### 3. 与 §12 路线图对齐

**强对齐。** 换手率因子是通往多个路线图里程碑的共用基础设施。

| §12 目标 | 对齐关系 | 对齐强度 |
|:---------|:---------|:---------|
| v1.5 Regime 切换检测准确率 ≥ 80% | 换手率是 Regime 分类的重要输入特征 | 🔴 强依赖 |
| Q4 容量评估完善（2026 Q3-Q4） | 流动性直接指标 | 🔴 强依赖 |
| 基金行为研究（基础版） | 换手率异常检测是判断大单行为的佐证 | 🟡 中等 |
| 多策略组合 | 流动性过滤 | 🟢 基础 |

**结论：因子2 是 v1.5 里程碑的必要基础能力之一，应纳入 Q3 优先交付清单。**

### 4. 复杂度评估：中等

| 评估项 | 结论 |
|:-------|:-----|
| 数据依赖 | 需使用 AKShare 原生 `换手率` 字段 + 清理 DB 中 0.0 值 |
| 计算逻辑 | 简单（读取列值），但数据层准备有运维工作量 |
| 风险等级 | 🟡 中 |
| 关键风险点 | DB 清理、AKShare 字段实测验证、两表同步 |
| 墨衡估算 | 1.5 人天 |
| 墨萱估算 | 2.0 人天（含测试修复 + DB 清理验证） |
| **玄知评级** | **中等（建议按 2.0 人天排期）** |

**对墨衡 vs 墨萱估算差异的裁决：采纳墨萱的 2.0 人天。**

理由：
1. **DB 清理不是"0 成本"的** — 墨萱指出的 3080 条 turnover_rate=0.0 记录清理需要执行 `UPDATE`、验证、备份，这是明确的工作量
2. **AKShare `换手率` 字段的实测验证** — 不是"改一行 column_map 就行"的事情。需要实测验证字段值域、单位（% vs 小数）、缺失率，与现有 data loading 流程对接
3. **两表（market_daily + stock_daily）同步** — 墨萱已指出两表都有 turnover_rate 列，补填逻辑需确保不遗漏
4. **测试验证** — 墨萱补充了 4 个测试模块（数据层验证+单元测试+集成测试+回滚测试），0.3-0.5 人天的工作量是合理的

### 5. 边界条件（墨衡+墨萱都未覆盖项）

墨萱已经做了非常详尽的遗漏分析。从玄知的研究价值角度补充 2 条：

**边界条件 1：换手率的市值相关性（size effect）。**

A 股中小盘标的的换手率系统性高于大盘蓝筹。例如工商银行日均换手率 ~0.2%，而某个小市值题材股日均换手率可达 10%+。在跨标的的换手率分析中：
- 直接使用绝对值会导致"选大小盘"的偏差
- **建议在 factor 文档中明确标注**：多标的比对时应使用换手率标准化值（Z-score 或 relative to historical mean）

**边界条件 2：新股/次换手率的波动特性。**

上市后 60 天内的换手率异常高（可达 50%+），与正常交易日的换手率不在同一量级。在 Regime 分类器中使用换手率作为输入时：
- **建议加入上市天数过滤器**：上市不足 60 天的标的，换手率不参与 Regime 分类计算
- **或对次新股做独立建模**：次新股和其他标的的换手率分布差异太大，混在一起会污染分类结果

### 6. 对墨萱补充的风险点的再确认

| 墨萱补充风险 | 玄知评价 | 处理建议 |
|:-------------|:---------|:---------|
| DB 中 turnover_rate=0.0 清理（⚠️ 高） | ✅ 认同，必须做 | 阶段一前置执行，补填前先 `SET turnover_rate=NULL` |
| 单位一致性验证 | ✅ 认同 | 添加验证断言 |
| AKShare 字段实测 | ✅ 认同 | 先跑 T-TC1 验证再推进 |
| 两表同步 | ✅ 认同 | 明确补填范围 |
| 测试工作量未纳入 | ✅ 认同 | 采纳 2.0 人天 |

### 对§9 优先级框架的影响

**不需要修改 §9 现有结构。** 这两个因子不是新的研究方向，而是已有方向的基础设施。它们归入 §9.2（基金行为研究）和 §9.3（Regime 动态策略）的铺路工作，不改变 §9 的方向优先级排序。

### 对§12 路线图的影响

**建议在§12 的"下一阶段"条目中补充：**
- **因子基础设施**：成交额均价 + 实际换手率 — 作为所有量价研究方向的公共数据基础设施，明确标记在 Q3 交付清单中

此条不是新增方向，而是对现有方向缺少基础设施的弥补。不改变路线图的里程碑结构。

### 最终裁决

**本项目可以放行。** 两个因子在研究和工程层面都具备充分实施条件。实施前提条件：

1. [ ] 墨萱补充的 DB 清理（turnover_rate=0.0）必须在阶段一执行
2. [ ] 墨萱设计的 4 个测试模块全部 PASS 后因子2 方可上线
3. [ ] 单位验证（amount/volume 值域检查）作为因子1 的必要验收条件
4. [ ] 换手率的 size effect 和次新股过滤需在文档中注明（作为边界条件提醒）
