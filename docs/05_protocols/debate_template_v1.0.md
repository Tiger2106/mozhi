# 辩论赛规则模板（辩论模板 v1.0）

**来源**: 右侧交易研讨会（2026-05-28 22:26 Owner制定）
**适用**: 墨枢团队三方观点分歧时，需结构化辩论的场景

---

## 一、触发条件

- 三方对某个方向/方案存在明显分歧
- Owner认为"各自说各自的"信息不对称，需要直面交锋
- 需要做优先级/方向决策前

---

## 二、辩论结构

### 角色
- **主持人（墨涵）**：计时、维持秩序、汇总结论
- **辩手 x3**：墨衡、墨萱、玄知
- **评审（Owner）**：旁观全部辩论后做最终决策

### 轮次

| 轮次 | 提问方 → 被问方 | 问题数上限 |
|:----:|:---------------:|:----------:|
| R1 | 墨萱 → 墨衡 | ≤3 |
| R2 | 玄知 → 墨衡 | ≤3 |
| R3 | 墨衡 → 墨萱 | ≤3 |
| R4 | 玄知 → 墨萱 | ≤3 |
| R5 | 墨衡 → 玄知 | ≤3 |
| R6 | 墨萱 → 玄知 | ≤3 |

### 核心规则

1. **每人被** 另外两人提问，每人 **回答** 另外两人的问题
2. **每人向另一个人提问不超过 3 个**
3. 每轮 = 提问方抛出问题 → 被问方在下一轮回答
4. 回答可以简短（单点回击），但必须正面回答问题
5. 禁止不回答、转移话题、说"这个问题下次讨论"
6. **每一问一答必须同步给不在场的第三方**，确保所有人都能听到全部对话

---

## 三、准备流程

### Step 1: 下发思路卡（主持人）
- 汇总三方各自的核心立场/提议/论点
- 每个角色一个独立思路卡，标明"你是哪个"
- 包含全量对方信息（让每个人知道对方会说什么）

### Step 2: 准备时间（5分钟，严格计时）
- 三方各自做三件事：
  1. 回顾自己立场，找出最弱环节（需防守）
  2. 准备回答对方可能的质问
  3. 准备向对方提 ≤3 个针锋相对的问题
- 准备期间禁止沟通
- **主持人**：倒计时结束后才可开始

### Step 3: 辩论开始
- 按 R1→R6 顺序进行
- 主持人宣布轮次切换
- 每轮将问题和回答同步给不在场的第三方

---

## 四、问题原则

| 好的问题 | 不好的问题 |
|----------|-----------|
| 针对对方论点中的逻辑缝隙 | 泛泛而问（"你怎么看？"） |
| 有具体切入点（数据/逻辑/假设） | 需要长篇大论的学术问题 |
| 对方必须回答才能过关 | "你觉得呢"式开放问题 |
| 暴露信息盲区或假设矛盾 | 承认自己不懂的问题 |

每个问题应做到：**对方听到就知道你攻击的是他的哪个论点**。
