## 四、建议灌入策略

### 推荐方案：**一次性灌入**

**理由：**
1. **数据量可控** — 即使全量（至多 12 万行），以 SQLite 而言只是数秒级的灌入
2. **API 限制** — Tushare 基础权限每分钟 200 次调用，一次 `pro.daily()` + `pro.adj_factor()` 取 39 只 × 5年 ≈ 78 次调用（39只 × 2 API），完全在限制内
3. **幂等性** — 可先 truncate 再灌入，或按 trade_date 更新已存在记录
4. **无质量问题** — 抽样验证 100% 可用，不存在需要分批修复的标的

**灌入建议步骤：**
1. 先处理 002013 替代方案（或用 600372.SH 替换）
2. 使用 `pro.daily()` 批量获取（建议 5 年或 3 年跨度）
3. 使用 `pro.adj_factor()` 同时获取复权因子
4. 写入 market_data.db
5. **灌入后验证：A50 全部 50 只成分股逐只核对（stock_basic 中的 A50 成分股全部有数据）**

**预估耗时：**
- API 调用耗时：~78 秒（39 只 × 2 API：daily + adj_factor）
- 数据写入 DB：~5 秒
- 总计：**< 2 分钟**
