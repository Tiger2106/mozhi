## Q1：截面IC要针对什么因子来计算？

**推荐方案：C. 多种因子同时测试**

截面IC的核心价值就是让数据说话——用相关性检验筛选有效因子，而不是依赖先验假设。目前`daily_factors`仅242条数据（3个因子），范围太窄。建议分两批：
- **第一批（T+7前完成）**：扩展`daily_factors`，纳入学术界验证有效的基础因子（波动率类、动量类、流动性类、价值类、质量类），每类2-3个，总计10-15个因子
- **第二批（后续迭代）**：由截面IC实证结果导向，保留有效因子、替换无效因子

## Q2：截面IC的频率？

**推荐方案：B. 周频**

- 日频噪声过大（T+1跳空、非同步交易等微观结构噪音会严重干扰截面相关性）
- 月频过于粗糙，且A50成分股调仓以季度为主，月频损失大量调仓期附近的信息
- 周频是截面IC的行业标准频率，2007-2026约950个截面足够做稳健性检验
- 作为补充：可在周频方案落地后，增加月频滚动窗口做对照

## Q3：Schema冻结——新建表还是扩展现有表？

**推荐方案：A. 新建独立截面IC结果表**

截面IC与时间序列IC的结构完全不同：
- 时间序列IC：每个stock×period一条记录，IC是单个标的当期值与未来收益的时序相关性
- 截面IC：每个factor×period一条记录，IC是当期该因子值与所有标的下期收益的截面秩相关性

**新表字段设计（初步）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTO | 主键 |
| `trade_date` | TEXT (YYYY-MM-DD) | 截面日期 |
| `factor_name` | TEXT | 因子名称（关联 daily_factors 的因子定义） |
| `ic_value` | REAL | 截面Pearson IC（因子值与下期收益的截面相关性） |
| `rank_ic` | REAL | Spearman秩IC（更稳健，推荐为主要指标） |
| `p_value` | REAL | 显著性p值 |
| `num_stocks` | INTEGER | 有效标的数（通常50） |
| `adjusted_ic` | REAL | 按市值/行业中性化后的IC |
| `created_at` | TEXT | 记录创建时间 |

**约束**：`(trade_date, factor_name)` 联合唯一索引，防止重复计算。
