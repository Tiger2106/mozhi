### 2. 回归测试套件（含 CI 嵌入）— 分层 CI 设计

**目的**：建立自动化回归安全网，防止修改引入回归缺陷，减少人工验证负担。这是回测系统质量保障的基石。

**内容**：
- **分层 CI 设计**（墨萱要求：PR 级 + 每日级）：
  - **PR 级（轻量）**：单元测试 + 代码风格检查 + import 完整性检查，5分钟内完成
  - **每日级（全面）**：PR 级 + 集成测试 + 基于历史数据的回测回归（选取 3-5 个典型策略），30 分钟内完成
- 选取典型回测场景编写回归用例（至少覆盖 5 种策略变体）
- 在 CI 系统（GitHub Actions / Jenkins）中配置流水线
- **关键约束**：CI 失败 → 阻止合并（墨萱要求）

**预期效果**：
- PR 提交后自动触发轻量测试，耗时 ≤ 5 分钟
- 每日凌晨自动触发全面回归，耗时 ≤ 30 分钟
- 测试覆盖率 ≥ 60%（语句覆盖）

**负责人**：墨衡
**预估工时**：3天（含 CI 配置）
**前后依赖**：依赖工作项 1（仓库就绪）。
**验收标准**：
- [x] PR 级流水线就位：提交即触发，5分钟内反馈
- [x] 每日级流水线就位：定时触发，30分钟内反馈
- [x] 基线回归用例 ≥ 123 个（VERIFY-001/002/003 全部纳入）
- [ ] 配置 CI 使其可自动执行基线回归
- [x] CI 失败时 **合并请求被自动阻止**（墨萱要求）
- [x] 回归结果通知渠道配置完成（飞书群通知）

### 3. 估值数据源摸底与激活（含数据新鲜度探针）

**目的**：摸清回测依赖的估值数据源现状（来源、格式、更新频率、可用性），建立数据新鲜度自动监控机制。

**内容**：
- 盘点所有外部估值数据源（股票、ETF、指数、期货等），记录每个数据源的：
  - 来源（Wind / iFinD / 量化平台 API / 自采数据）
  - 更新频率（每日收盘后 / 实时 / T+1）
  - 历史覆盖深度
  - 已知质量问题（缺失、异常值、停牌未剔除等）
- **数据新鲜度探针**（玄知要求）：编写探针脚本，定时检查数据源是否有新数据到达
  - 探针阈值：同一 key 数据超过配置小时数未更新 → 告警（具体阈值由**玄知在摸底报告中签字确认**）
  - 数据源类型阈值参考（待玄知确认）：
    - 日频数据（收盘价、日因子）：24 小时未更新 → WARN，48 小时未更新 → ALERT
    - 日内高频数据（分钟线、Tick）：4 小时未更新 → WARN，8 小时未更新 → ALERT
    - 基本面数据（财报、财报预告等）：T+3 工作日未更新 → WARN，T+5 工作日未更新 → ALERT
    - 实时行情（含期指）：30 分钟未更新 → WARN，2 小时未更新 → ALERT
- 对已在使用的数据源进行激活验证：跑一次全量回测，确认数据可用
- 输出《估值数据源摸底报告》

**预期效果**：
- 所有数据源清晰建档，无盲区
- 新鲜度探针每天自动运行，数据延迟超时自动告警
- 已激活数据源可支撑完整回测

**负责人**：玄知（数据）+ 墨衡（探针开发）
**预估工时**：3天
**前后依赖**：无（可与工作项 1/2 平行）
**验收标准**：
- [x] 数据源摸底报告完成，覆盖 ≥ 90% 在用数据源
- [x] 新鲜度探针部署运行，触发告警阈值合理
- [x] 至少 1 次全量回测验证通过

- **第一级：端到端样本验证**

- 将两级验证编排为可复用的验证管线

**预期效果**：
- 每次回测运行后自动执行两级验证
- 黄金样本校验一致性 ≥ 99.5%
- 输出校验失败自动标记并阻断进入下游

**负责人**：墨衡
**预估工时**：3天
**前后依赖**：依赖工作项 1（仓库）和部分依赖工作项 2（基础用例框架）
**验收标准**：
- [x] 3 个黄金样本验证用例通过
- [x] 输出校验规则 ≥ 5 条
- [x] 两级验证可独立运行，也可串联执行

### 6. Pipeline 异常路径测试 / 空跑（P2）

**目的**：测试回测流水线在非正常情况下的行为，确保异常发生时系统能优雅处理，不吞没错误也不崩溃（玄知新增要求）。

**内容**：
- 定义异常路径集合：
  - **数据异常**：数据源不可用、字段缺失、数值异常
  - **参数异常**：超范围参数、类型错误
  - **并发异常**：重复提交、资源竞争
  - **空跑场景**：空持仓、空数据区间、空策略文件
- 为每个场景编写测试用例，记录预期行为
- 验证系统在上述情况下是否：
  - 抛出明确错误信息
  - 落盘错误日志 + 告警
  - 不进入异常计算结果
- 修复测试中发现的缺陷

**预期效果**：异常场景测试覆盖率 ≥ 80%，空跑场景一律优雅退出。

**负责人**：墨衡
**预估工时**：2天
**前后依赖**：依赖工作项 2（测试框架就位）
**与 Week 2 的关系**：异常路径测试用例设计可在 Week 2 提前进行（与数据血缘设计并行），Week 3 集中执行验证。Week 2 为设计阶段，Week 3 为执行+修复阶段。两者为阶段性延续关系，非独立并行项。
**验收标准**：
- [x] 异常路径测试用例 ≥ 8 个
- [x] 空跑场景测试用例 ≥ 3 个
- [x] 所有异常场景均产出明确错误信息 + 日志落盘
- [x] 空跑场景优雅退出，无崩溃
