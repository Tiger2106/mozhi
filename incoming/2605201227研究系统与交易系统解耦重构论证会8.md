# Freeze Review 必查五项

---

## 1. 是否出现协议重复定义

例如：

- REJECTED 在 §3.1 和 §3.2 是否一致

- extras 红线是否与例外冲突

- V-13 是否与旧验证规则矛盾

---

## 2. 是否出现“软性措辞”

必须避免：

```text
建议
尽量
通常
推荐
一般情况下
```

协议必须：

```text
MUST
MUST NOT
SHALL
SHALL NOT
```

否则未来无法裁决。

---

## 3. 是否出现“跨层越权”

例如：

Signal Protocol 里出现：

- Portfolio逻辑

- Execution实现

- Knowledge TTL细节

这些都不应该出现。

---

## 4. 是否出现“未来字段污染”

尤其检查：

```text
extras.*
```

必须确认：

- 哪些只是命名空间

- 哪些是正式字段

- 哪些只是预留

否则未来会出现：

```json
{
  "extras": {
    "random_field": "..."
  }
}
```

---

## 5. v1 与 v1.1 的升级边界是否清晰

这是最容易埋雷的。

现在：

```text
v1.0 = warning
v1.1 = reject
```

这里必须明确：

- 哪些字段在 v1.1 变硬性

- 老 Consumer 如何降级

- 是否允许 mixed mode

否则上线后会出现：

```text
研究系统能发
交易系统不能收
```
